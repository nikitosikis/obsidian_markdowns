
Очистка данных (Data Cleaning) представляет процесс выявления и исправления ошибок, несоответствий и неточностей в наборе данных. Он включает в себя различные методы для обеспечения точности, надежности и пригодности данных для анализа. Очистка данных является важным этапом в рабочем процессе анализа данных, поскольку качество выводов и заключений, полученных в результате анализа данных, в значительной степени зависит от качества исходных данных. Поскольку исходные данные, собранные из различных источников, могут содержать ошибки, вызванные человеческим фактором, системными ошибками или проблемами передачи данных. Кроме того, данные могут содержать пропущенные значения, дублирующиеся записи или несоответствия, которые могут исказить результаты анализа, если их не устранить должным образом. Очистка данных направлена ​​на исправление этих проблем и создание чистого и достоверного набора данных, который можно использовать для более точного и содержательного анализа.

Очистка данных гарантирует исправление ошибок и несоответствий, помогает уменьшить предвзятость и ошибки, удаляtn ненужный шум и нерелевантную информацию, что приводит к более точным и надежным результатам анализа.

В то же время надо понимать, что очистка данных может быть сложной и трудоемкой задачей, и в процессе возникают различные проблемы. Например, в наборах данных могут отсутствовать некоторые значения по различным причинам, что может повлиять на результаты анализа, если это не будет должным образом учтено. Либо могут иметься дублирующиеся записи, что может исказить анализ и привести к неверным выводам.

## Удаление пропущенных данных

Пропущенные данные представляют отсутствие значений в определенных полях или столбцах набора данных. Это может происходить по различным причинам, таким как отсутствие сбора данных, повреждение данных во время хранения или передачи, или человеческие ошибки при вводе данных. На уровне кода подобные пропущенные значения обычно представлены либо константой `nan` из numpy или стандартным значением `None` в Python. Работа с пропущенными данными необходима для предотвращения предвзятых или неточных результатов анализа. Для обнаружения пропущенных данных в Pandas применяется ряд функций: `isna()`, `isnull()` и `notnull()`, которые рассматривались в статье Фильтрация значений NULL или пропущенных значений. Поэтому в этой статье мы сосредоточимся на удалении пропущенных значений.

### Удаление строк с пропущенными значениями и dropna()

Метод dropna() в Pandas используется для удаления строк с пропущенными значениями из DataFrame. По умолчанию он удаляет все строки, содержащие хотя бы одно пропущенное значение. Этот метод помогает устранить неполные или недостоверные данные, которые могут негативно повлиять на анализ данных. Синтаксис метода:

```python
import pandas as pd
DataFrame.dropna(axis=0, how='any', inplace=False)
```

Метод принимает следующие параметры:

- `axis`: задает ось, вдоль которой следует удалять строки. По умолчанию установлено значение 0, что означает удаление строк.
    
- `how`: определяет критерии удаления строк. Может принимать значения "any" (по умолчанию) или "all". "any" удаляет строки, в которых есть хотя бы одно пропущенное значение, а "all" удаляет строки, где все значения пропущены.
    
- `inplace`: если равно `True`, изменяет DataFrame на месте и возвращает `None`. Если равно `False` (по умолчанию), возвращает новый DataFrame с удаленными строками.
    

Используем функцию `dropna()` для удаления строк с пропущенными значениями:

```python
import numpy as np
data = {
'A': [1, 2, np.nan, 4, None],
'B': [None, 6,7,8, np.nan]
}
df = pd.DataFrame(data)
cleared_df = df.dropna()
>>> print(df)
>>> print(cleared_df)
     A    B
0  1.0  NaN
1  2.0  6.0
2  NaN  7.0
3  4.0  8.0
4  NaN  NaN

     A    B
1  2.0  6.0
3  4.0  8.0
```

В данном случае функция `dropna()` по умолчанию не изменяет исходный DataFrame - df. Но если надо изменить df на месте, можно в функцию `dropna()` передать аргумент `inplace=True`.

### Удаление данных в определенных столбцах

При использовании функции `dropna()` с подмножеством столбцов можно удалять строки, которые содержат пропущенные значения только в определенных столбцах, оставляя остальные столбцы нетронутыми. Это может быть полезно, если надо сосредоточиться на определенных столбцах для анализа и обрабатывать пропущенные данные в этих столбцах иначе, чем в других. Например:

```python
data = {
"A": [1, 2, np.nan, 4, None],
"B": [None, 6, 7, 8, np.nan],
"C": [9, None, 11, 12, 13]
}

df = pd.DataFrame(data)

cleared_df = df.dropna(subset=["A", "B"])
>>> print(df)
>>> print(cleared_df)
     A    B     C
0  1.0  NaN   9.0
1  2.0  6.0   NaN
2  NaN  7.0  11.0
3  4.0  8.0  12.0
4  NaN  NaN  13.0

     A    B     C
1  2.0  6.0   NaN
3  4.0  8.0  12.0
```

В этом примере используем функцию `dropna()` с параметром `subset`, чтобы указать, что мы хотим учитывать только столбцы "A" и "B" для удаления строк с пропущенными значениями. А строки с пропущенными значениями в столбце "C" остаются без изменений. Полученный DataFrame - cleared_df содержит только строки с полными данными в столбцах "A" и "B".

# Заполнение пропущенных значений константой, средним значением и медианой

Функция fillna() в Pandas используется для заполнения пропущенных значений (NaN / None) в DataFrame или Series определенными значениями. Эта функция предоставляет несколько способов обработки пропущенных данных в зависимости от характера данных и требований к анализу. Рассмотрим различные примеры использования функции `fillna()` с различными типами данных и стратегиями заполнения пропущенных значений. Результатом этой функции является DataFrame той же формы, что и исходный DataFrame, к которому применялась эта функция.

### Заполнение пропущенных данных константой

Один из подходов при работе с пропущенными данными заключается в заполнении этих пропущенных значений некоторой константой. Это означает, что все пропущенные ячейки будут заменены одним и тем же фиксированным значением. Выбор константы зависит от контекста и характера данных. К распространенным вариантам относятся 0, -1 или определенное значение, заведомо выходящее за пределы допустимого диапазона данных.

В то же время надо осторожно относиться к заполнению пропущенных значений константой, поскольку это может потенциально внести искажения в статистический анализ, особенно если значение константы (например, 0) имеет специфическое значение в контексте данных. Поэтому крайне важно тщательно учитывать последствия заполнения пропущенных значений константой и выбирать подходящее значение константы, исходя из характера данных и требований анализа.

Рассмотрим пример применения функции:

```python
data = {
'A': [1, 2, np.nan, 4, None],
'B': [None, 6,7,8, np.nan]
}
df = pd.DataFrame(data)
filled_df = df.fillna(0)
>>> print(filled_df)
     A    B
0  1.0  0.0
1  2.0  6.0
2  0.0  7.0
3  4.0  8.0
4  0.0  0.0
```

Здесь у нас есть DataFrame - df с двумя столбцами "A" и "B", которые содержат пропущенные значения (`np.nan/None`). После применения функции `fillna()` все подобные пропущенные значения заменяются константным значением 0.

### Заполнение средним значением или медианой

При работе с пропущенными числовыми данными альтернативный подход заключается в заполнении этих пропущенных значений средним значением или медианой непропущенных значений в том же столбце. Этот метод полезен, когда предполагается, что пропущенные значения имеют центральную тенденцию, аналогичную существующим точкам данных в этом столбце. Используя среднее значение или медиану, мы стремимся минимизировать влияние пропущенных данных на общие статистические свойства набора данных.

Использование среднего значения подходит, когда распределение данных приблизительно симметрично и не содержит значительных выбросов (Выброс (outlier) - это экстремальные значения во входных данных, которые находятся далеко за пределами других наблюдений и которые сильно отличаются от общего распределения). С другой стороны, если распределение данных асимметрично или содержит выбросы, медиана может быть более надежным выбором для заполнения пропущенных значений.

#### Заполнение средним значением

Рассмотрим следующий пример:

```python
data = {
'A': [1, 2, np.nan, 4, None],
'B': [None, 6,7,8, np.nan]
}
df = pd.DataFrame(data)
filled_df = df.copy()
filled_df['A'] = df['A'].fillna(df['A'].mean())
>>> print(filled_df)
          A    B
0  1.000000  NaN
1  2.000000  6.0
2  2.333333  7.0
3  4.000000  8.0
4  2.333333  NaN
```

Для вычисления среднего значения всех непропущенных значений в столбце "A" используется метод `df["A"].mean()`. Затем полученное среднее значение передается в метод `fillna()`, который заполняет пропущенные значения в столбце "A". здесь мы используем передавая ему среднее значение непропущенных значений из этого же столбца.
По консольному выводу мы видим, что среднее значение по столбцу "A" - 2.333333. Этот метод гарантирует, что пропущенные значения в столбце "A" будут заменены значением, представляющим среднее значение существующих точек данных в этом столбце. Это помогает сохранить центральную тенденцию данных и обеспечивает сохранение общих статистических свойств.
#### Заполнение медианой

Рассмотрим следующий пример:

```python
data = {
'A': [1, 2, np.nan, 4, None],
'B': [None, 6,7,8, np.nan]
}
df = pd.DataFrame(data)
filled_df = df.copy()
filled_df['B'] = df['B'].fillna(df['B'].median())
>>> print(filled_df)
     A    B
0  1.0  7.0
1  2.0  6.0
2  NaN  7.0
3  4.0  8.0
4  NaN  7.0
```

Здесь заполненяем медианой пропущенные значения в столбце "B". Для получения медианы применяется метод `df["B"].median()`, который вычисляет медиану всех непропущенных значений в столбце "B". А затем метод `fillna()` заполняет пропущенные значения в "B" вычисленной медианой.
Заполнение медианой представляет надежный подход, особенно когда распределение данных асимметрично или содержит выбросы, поскольку медиана менее чувствительна к экстремальным значениям.

# Заполнение с помощью конкретной категории

При работе с отсутствующими категориальными данными один из распространенных подходов - заполнение пропущенных значений конкретной категорией или меткой. Этот метод подходит, когда предполагается, что пропущенные значения представляют собой отдельную категорию, отличную от существующих категорий в наборе данных. Заполняя пропущенные значения конкретной категорией, мы избегаем внесения смещения в данные и сохраняем категориальный характер переменной.

При работе с отсутствующими категориальными данными существует два распространенных подхода к заполнению пропущенных значений: заполнение с помощью конкретной категории и заполнение с помощью моды.

### Заполнение с помощью конкретной категории

Этот метод предполагает заполнение пропущенных категориальных значений заранее определенной категорией или меткой, которая обозначает отсутствие допустимой категории. Он подходит, когда предполагается, что пропущенные значения должны иметь значение, отличное от существующих категорий в наборе данных. Распространенные варианты для конкретной категории включают такие метки, как "Unknown" ("Неизвестно"), "Not Applicable" ("Неприменимо"), "Other" ("Другое") или что-то наподобие, что четко указывает на отсутствие допустимой категории. Используя определенную категорию, мы гарантируем, что пропущенные значения будут обрабатываться отдельно в последующих анализах и избежим внесения искажений в данные. Например:

```python
data = {
'A': [1, 2, np.nan, 4, None],
'B': [None, 6,7,8, np.nan]
}
df = pd.DataFrame(data)
df = df.fillna('Unknown')

1      2.0      6.0
2  Unknown      7.0
3      4.0      8.0
4  Unknown  Unknown
```

Мода (mode) представляет собой наиболее часто встречающуюся категорию. При использовании моды пропущенные значения заполняются категорией, которая наиболее часто встречается среди непропущенных значений в том же столбца. Этот метод подходит, когда считается, что пропущенные значения похожи на наиболее распространенную категорию в наборе данных. Заполнение модой помогает сохранить общее распределение категорий. Например:

data = {

# Обратное и прямое заполнение пропущенных значений

При работе с пропущенными данными для заполнения пропущенных значений также могут использоваться еще два метода: обратное заполнение (backfill) и прямое заполнение (forward fill).

### Прямое заполнение (forward fill)

Прямое заполнение предполагает заполнение пропущенных значений самыми последними доступными данными. При обнаружении пропущенного значения метод прямого заполнения ищет самое последнее непропущенное значение перед точкой пропущенных данных и использует его для заполнения пропущенного значения. Этот процесс продолжается до тех пор, пока не будут заполнены все пропущенные значения или пока не будет обнаружено другое пропущенное значение.

Прямое заполнение особенно полезно при работе с временными рядами или любыми последовательными данными, где наблюдения имеют естественный порядок. В таких случаях заполнение пропущенных значений самыми последними доступными данными часто является разумным подходом, поскольку это помогает поддерживать временную непрерывность набора данных.

Для прямого заполнения применяется метод ffill():

```python
data = {
"A": [1, 2, np.nan, 4, None],
"B": [None, 6, 7, 8, np.nan]
}
df = pd.DataFrame(data)
# Прямое заполнение пропущенных значений
filled_df = df.ffill()
>>> print(df)
>>> print(filled_df)
     A    B
0  1.0  NaN
1  2.0  6.0
2  NaN  7.0
3  4.0  8.0
4  NaN  NaN

     A    B
0  1.0  NaN
1  2.0  6.0
2  2.0  7.0
3  4.0  8.0
4  4.0  8.0
```

В выводе видно, что пропущенные значения в столбцах "A" и "B" были заполнены последними непропущенными значениями. Например, пропущенное значение под индексом 2 столбца "A" было заполнено значением 2.0, которое является последним непропущенным значением перед индексом 2. Заполнение остальных пропущенных значений было аналогично за исключением значения с индексом 0 в столбце "B" - оно по-прежнему содержит пропущенное значение (NaN). Потому что перед ним нет непропущенного значения, которое можно было бы использовать для заполнения. В подобных случаях прямое заполнение не может быть применено для заполнения этих пропущенных значений, поскольку нет предыдущих данных, которые можно было бы использовать для заполнения. И в этом случае в дело вступает обратное заполнение.

### Обратное заполнение (backfill)

Обратное заполнение (backfill) используется для заполнения пропущенных значений следующим НЕпропущенным значением. То есть заполнение идет в обратном порядке. Этот процесс продолжается до тех пор, пока не будут заполнены все пропущенные значения или пока не будет обнаружено другое пропущенное значение.

Для выполнения обратного распространения используется метод bfill():

```python
data = {
"A": [1, 2, np.nan, 4, None],
"B": [None, 6, 7, 8, np.nan]
}
df = pd.DataFrame(data)
# Обратное заполнение пропущенных значений
filled_df = df.bfill()
>>> print(df)
>>> print(filled_df)
     A    B
0  1.0  NaN
1  2.0  6.0
2  NaN  7.0
3  4.0  8.0
4  NaN  NaN

     A    B
0  1.0  6.0
1  2.0  6.0
2  4.0  7.0
3  4.0  8.0
4  NaN  NaN
```


Из вывода видно, что значение с индексом 2 в стобце "A" заполняется следующим значением - числом 4.0, а пропущенное значение с индексом 0 в столбце "B" заполняется следюущим значением - числом 6.0. Однако пропущенные значения с индексом 4 в обоих столбцах остаются пропущенными, так как после них нет никаких непропущенных значений. В этом и состоит недостаток обратного распространения. Поэтому в идеальном случае было бы лучше сочетать оба способа заполнения - прямое и обратное:

```python
data = {
"A": [1, 2, np.nan, 4, None],
"B": [None, 6, 7, 8, np.nan]
}
df = pd.DataFrame(data)
# Прямое заполнение пропущенных значений
filled_df = df.ffill()
# Обратное заполнение пропущенных значений
filled_df = filled_df.bfill()
>>> print(df)
>>> print(filled_df)
     A    B
0  1.0  NaN
1  2.0  6.0
2  NaN  7.0
3  4.0  8.0
4  NaN  NaN

     A    B
0  1.0  6.0
1  2.0  6.0
2  2.0  7.0
3  4.0  8.0
4  4.0  8.0
```

# Интерполяция для обработки пропущенных данных

Интерполяция представляет метод, используемый для оценки пропущенных значений в наборе данных на основе существующих точек данных. Он особенно полезен для обработки пропущенных данных во временных рядах или последовательных данных, где пропущенные значения могут возникать по различным причинам, таким как сбои датчиков или пробелы в сборе данных. Методы интерполяции вычисляют пропущенные значения, рассматривая значения соседних точек данных, делая обоснованное предположение о том, какими могут быть пропущенные значения, то есть интерполируют существующие значения на пропущенные. Есть разные способы интерполяции, рассмотрим некоторые ее виды.

### Линейная интерполяция

Одним из распространенных методов интерполяции является линейная интерполяция, где пропущенные значения оцениваются на основе прямой линии, соединяющей две соседние точки данных. Он основан на предположении, что связь между точками данных линейна, то есть изменение между последовательными точками данных постоянно.

Для выполнения линейной интерполяции применяется функция interpolate(), в которую передается аргумент `method="linear"`:

```python
filled_df = df.interpolate(method='Linear')
>>> print(df)
>>> print(filled_df)
     A    B
0  1.0  NaN
1  2.0  6.0
2  NaN  7.0
3  4.0  8.0
4  NaN  NaN

     A    B
0  1.0  NaN
1  2.0  6.0
2  3.0  7.0
3  4.0  8.0
4  4.0  8.0
```

Для пропущенного значения в столбце A с индексом 2 Pandas выполняет линейную интерполяцию между точками данных (2.0, 6.0) и (4.0, 8.0), оценивая значение как 3.0. Другое пропущенное значение в столбце A с индексом 4 заполняется значением 4.0, поскольку нет других точек данных для выполнения интерполяции.

Аналогично пропущенное значение в столбце B с индексом 4 заполняется числом 8.0, поскольку нет других точек данных для выполнения интерполяции. А вот пропущенное значение с индексом 0 (`B[0]`) остается незаполненным.

Дело в том, что метод `interpolate()` по умолчанию не может интерполировать первый элемент, так как для линейной интерполяции нужно значение до и после пропуска, и работает только "вперед", заполняя пропуски на основе следующих значений (неявно используется аргумент `limit_direction="forward"`)

В качестве решения мы можем либо сочетать интерполяцию с прямым/обратным заполнением, либо явным образом указать методу `interpolate()`, что надо работать не только "вперед", но и "назад" - для этого применяется аргумент `limit_direction="both"`:

```python
filled_df = df.interpolate(method='Linear', limit_direction='both')
>>> print(filled_df)
     A    B
0  1.0  6.0
1  2.0  6.0
2  3.0  7.0
3  4.0  8.0
4  4.0  8.0
```

### Интерполяция по времени

Интерполяция по времени (Time-Based Interpolation) представляет специализированный тип интерполяции, используемый для временных рядов - диапазонов временных точек, где пропущенные значения оцениваются на основе временного индекса точек данных.

Для примера сначала рассмотрим, как создать временной ряд:

```python
time_index = pd.date_range(start="2025-01-01", periods=5)
time_series = pd.Series([10, np.nan, 30, np.nan, 50], index=time_index)
>>> print(time_series)
2025-01-01    10.0
2025-01-02     NaN
2025-01-03    30.0
2025-01-04     NaN
2025-01-05    50.0
Freq: D, dtype: float64
```

Для создания временного ряда в библиотеке Pandas применяется метод `date_range()`, который генерирует индекс DatetimeIndex с фиксированной частотой (последовательность дат и времени). Ее ключевые параметры:

- `start`: левая граница (начальная дата) для генерации дат.
    
- `end`: правая граница (конечная дата) для генерации дат.
    
- `periods`: количество периодов (дат/временных точек) для генерации.
    
- `freq`: частота генерируемых дат, например, "D" (ежедневно), "H" (ежечасно), "M" (конец месяца), "B" (рабочий день) и т. д. По умолчанию - "D".
    

Здесь создается временной ряд, в котором временная метка первого элемента представляет дату "2025-01-01" (1 января). И всего генерируется пять подобных точек, которые соответствуют дням - от 1 до 5 января (так как для параметра `freq` значение по умолчанию - "D")

 для примера создаем объект Series - time_series, который использует сгенерированный индекс на основе меток даты/времени. И также для примера определяем в этой серии 2 пропущенных значения (для меток 2025-01-02 и 2025-01-04)

Теперь выполним на примере этой серии интерполяцию по времени. В Pandas для этого использется метод interpolate() с параметром method="time". Этот метод учитывает временной индекс и соответствующим образом выполняет интерполяцию:

```python
series_interpolated = time_series.interpolate(method='time')
>>> print(series_interpolated)
2025-01-01    10.0
2025-01-02    20.0
2025-01-03    30.0
2025-01-04    40.0
2025-01-05    50.0
Freq: D, dtype: float64
```

Здесь для пропущенного значения с меткой "2025-01-02" pandas выполняет интерполяцию по времени между доступными точками данных "2025-01-01" (10.0) и "2025-01-03" (30.0) и оценивает пропущенное значение как 20.0. Для отсутствующего значения в точке "2025-01-04" Pandas выполняет интерполяцию по времени между доступными точками данных "2025-01-03" (30.0) и "2023-01-05" (50.0) и оценивает пропущенное значение как 40.0. В итоге после интерполяции по времени данные представляют собой непрерывный временной ряд, в котором отсутствующие значения оцениваются на основе временных меток.

# Обработка дубликатов

Дубликаты в данных представляют идентичные или почти идентичные записи, которые встречаются в наборе данных более одного раза. Дублирующиеся данные возникают, когда две или более записи в наборе данных имеют одинаковые значения во всех или большинстве своих столбцов. Эти дубликаты могут быть либо точными копиями, либо очень похожими записями с небольшими вариациями. При определенных требованиях дубликаты могут привести к искаженному анализу, неточной статистике и предвзятым результатам. Поэтому может возникнуть необходимость выявлять и удалять дублирующиеся данные из наборов данных, чтобы обеспечить целостность данных и надежность анализа.

Для обнаружения дублирующихся строк в DataFrame применяется функция duplicated(). Эта функция помогает определить, какие строки являются дубликатами, а какие - уникальными на основе значений в одном или нескольких столбцах. Функция возвращает логический объект типа Series, где каждый элемент соответствует строке в DataFrame. Значение равно `True` для дубликатов и `False` для недублирующихся, уникальных строк.

Причем поведение функции `duplicated()` по умолчанию заключается в том, чтобы пометить все дубликаты после первого появления как `True`. Что означает, что первое появление не считается дубликатом. Однако это поведение можно настроить с помощью параметра `keep`:

```python
DataFrame.duplicated(subset=None, keep='first')
```

Параметры функции:

- `subset`: метка столбца или последовательность меток, которая однозначно идентифицирует строки для рассмотрения на предмет дубликатов. Если `subset` не указан, функция просматривает все столбцы для обнаружения дубликатов.
    
- `keep`: определяет, какие дубликаты (если таковые имеются) пометить как `True`. И здесь есть следующие варианты:
    
    - `"first"` (по умолчанию): пометить дубликаты кроме первого появления.
        
    - `"last"`: пометить дубликаты кроме последнего вхождения.
        
    - `False`: пометитьвсе дубликаты как `True`.
        

Результатом функции является объект Series, где каждый элемент указывает, является ли соответствующая строка дубликатом (`True`) или нет (`False`). Посмотрим на примере. Сначала посмотрим, как найти строки-дубликаты:

```python
data = {
"City": ["Москва", "Санкт-Петербург", "Новосибирск", "Москва", "Казань", "Красноярск", "Волгоград", "Новосибирск"],
"Population": [13274, 5653, 1637, 13274, 1330, 1212, 1012, 1637],
"Time zone": ["GMT+3", "GMT+3","GMT+7","GMT+3","GMT+3","GMT+7","GMT+3","GMT+7"]
}

df = pd.DataFrame(data)
duplicates = df.duplicated()
>>> print(df)
>>> print(duplicates)
              City  Population Time zone
0           Москва       13274     GMT+3
1  Санкт-Петербург        5653     GMT+3
2      Новосибирск        1637     GMT+7
3           Москва       13274     GMT+3
4           Казань        1330     GMT+3
5       Красноярск        1212     GMT+7
6        Волгоград        1012     GMT+3
7      Новосибирск        1637     GMT+7
0    False
1    False
2    False
3     True
4    False
5    False
6    False
7     True
dtype: bool
```

В данном случае строки 3 и 7 содержат полные дубликаты-строки, поэтому в выходных данных они помечены как True. Используя полученный объект Series в качестве индекса, мы можем отфильтровать дубликаты:

```python
duplicates = df.duplictated()
duplicate_rows = df[duplicates]
>>> print(duplicate_rows)
          City  Population Time zone
3       Москва       13274     GMT+3
7  Новосибирск        1637     GMT+7
```

По умолчанию функция duplicated() использует всю строку для обнаружения дубликатов. Но с помощью параметра `subset` можно указать столбцы, относительно которых будут рассчитываться дубликаты: дублями будут считаться только те строки, которые в данных столбцах имеют одинаковые значения. Например, если мы хотим для обнаружения дубликатов учитывать только столбцы "City" и "Population":

```python
duplicates = df.duplicated(subset=['City', 'Population'])
```

### Удаление дублирующихся строк

Выше мы посмотрели, как выявить дубли в наборе данных, но остается вопрос: как от них избавиться? Для этого применяется функция drop_duplicates(). Она находит и удаляет дублирующиеся строки из DataFrame, сохраняя по умолчанию только первое вхождение каждой дублирующейся строки. Это означает, что первое вхождение набора дубликатов считается недублирующимся, а последующие дубликаты удаляются.

Эта функция имеет следующий синтаксис:

```python
DataFrame.drop_duplicates(subset=None, keep='first', inplace=False)
```

Параметры функции:

- `subset`: метка столбца или последовательность меток, которая однозначно идентифицирует строки для проверки на наличие дубликатов. Если `subset` не указан, функция рассматривает все столбцы для обнаружения дубликатов.
    
- `keep`: определяет, какие дубликаты (если таковые имеются) следует сохранить. Может принимать следующие значения:
    
    - `"first"`(по умолчанию): сохраняет первое вхождение и удаляет последующие дубликаты.
        
    - `"last"`: сохраняет последнее вхождение и удаляет предыдущие дубликаты.
        
    - `False`: удаляет все дубликаты.
        
- `inplace`: если равно `True`, изменяет текущий объект DataFrame, к которому функция и применяется, и возвращает `None`. Если равно `False` (по умолчанию), возвращает новый DataFrame с удаленными дубликатами.
    

Для примера возьмем следующий скрипт:

```python
data = {
"City": ["Москва", "Санкт-Петербург", "Новосибирск", "Москва", "Казань", "Красноярск", "Волгоград", "Новосибирск"],
"Population": [13274, 5653, 1637, 13274, 1330, 1212, 1012, 1637],
"Time zone": ["GMT+3", "GMT+3","GMT+7","GMT+3","GMT+3","GMT+7","GMT+3","GMT+7"]
}

df = pd.DataFrame(data)
df_cleared = df.drop_duplicates()
>>> print(df)
>>> print(df_cleared)
              City  Population Time zone
0           Москва       13274     GMT+3
1  Санкт-Петербург        5653     GMT+3
2      Новосибирск        1637     GMT+7
3           Москва       13274     GMT+3
4           Казань        1330     GMT+3
5       Красноярск        1212     GMT+7
6        Волгоград        1012     GMT+3
7      Новосибирск        1637     GMT+7

              City  Population Time zone
0           Москва       13274     GMT+3
1  Санкт-Петербург        5653     GMT+3
2      Новосибирск        1637     GMT+7
4           Казань        1330     GMT+3
5       Красноярск        1212     GMT+7
6        Волгоград        1012     GMT+3
```

Здесь строки с индексами 3 и 7 являются дубликатами, поэтому после применения функции `drop_duplicates()` эти дубликаты удаляются, и новый DataFrame - df_cleared содержит только уникальные записи.
