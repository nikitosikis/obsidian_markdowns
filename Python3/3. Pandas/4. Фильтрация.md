Фильтрация представляет одну из ключевых задач при работе с данными, которая позволяет извлекать определенные подмножества данных на основе определенных условий. Фильтрация данных служит фундаментальным этапом предварительной обработки данных для уточнения и адаптации набора данных к требованиям проводимого анализа или исследования. Применяя фильтры, аналитики могут сосредоточиться на данных, имеющих отношение к их конкретному исследовательскому вопросу, что упрощает процесс анализа и повышает точность результатов.

Наборы данных часто содержат большое количество строк и столбцов. Фильтрация же позволяет уменьшить размер данных Фильтрация данных позволяет, исключить ненужную или менее релевантную и актуальную информацию, исключить аномалии и выбросы. В результате чего получается меньший, более управляемый и более релевантный набор данных. Это повышает эффективность анализа данных и снижает вычислительные затраты.

Фильтрация данных обычно проводится для табличных структур данных, таких как наборы данных **DataFrame**, где строки представляют наблюдения, а столбцы - переменные. Аналитики могут фильтровать строки или столбцы на основе различных условий, таких как числовые сравнения, сопоставление строк, диапазоны дат или комбинации нескольких условий.

Библиотека Pandas предоставляет богатый набор возможностей для фильтрации данных:

- Условное индексирование: Pandas позволяет фильтровать строки на основе определенных условий, применяемых к значениям столбцов. Это достигается путем создания логических масок, указывающих, выполняется ли определенное условие для каждой строки.
    
- Логические операторы: такие операторы как & (and) и | (or) позволяют комбинировать условия для создания сложных критериев фильтрации, что позволяет извлекать подмножества данных, которые отвечают сложным требованиям.
    
- Числовая фильтрация: Pandas поддерживает фильтрацию строк на основе числовых условий, например, фильтрацию строк со значениями больше, меньше или в пределах определенного диапазона.
    
- Фильтрация по дате и времени: в Pandas мы можем фильтровать данные на основе условий по дате и времени. Это особенно полезно при работе с временными рядами.
    
- Фильтрация по нулевым значениям: Pandas предоставляет методы для фильтрации строк или столбцов с пропущенными значениями (NaN) или непропущенными значениями, что позволяет эффективно обрабатывать неполные данные.
    
- Запросы с помощью метода query(): Pandas предлагает метод query(), который позволяет писать SQL-подобные запросы для фильтрации данных на основе определенных условий.
    

### Фильтрация DataFrame на основе столбцов

Самая базовая форма фильтрации в DataFrame представляет фильтрация по столбцам, где мы выбираем определенные столбцы из DataFrame. Фильтрация DataFrame на основе меток определенного столбца или набора столбцов означает выбор подмножества данных из DataFrame, в которое включены только указанные столбцы. Этот процесс включает в себя применение условий или отбора к меткам столбцов для сохранения нужных столбцов и создания нового объекта DataFrame или объекта Series, который содержит только отфильтрованные данные.

```python
data = {
```

```python
columns = ['City', 'Time Zone']
```

# Фильтрация с помощью логических операторов

В Pandas для фильтрации строк в DataFrame применяются логические выражения. Логические выражения включают операции сравнения и логические операции, результатом которых являются значения `True` или `False`.

Операторы сравнения, как и в целом в Python, используются для сравнения значений и оценки условий и возвращают логическое значение на основе результата сравнения:

- Равенство `(==)`: возвращает `True`, если операнды равны.
    
- Неравенство (!=): возвращает `True`, если операнды не равны.
    
- Больше (>): возвращает `True`, если левый операнд больше правого операнда.
    
- Меньше (<): возвращает `True`, если левый операнд меньше правого операнда.
    
- Больше или равно (>=): возвращает `True`, если левый операнд больше или равен правому операнду.
    
- Меньше или равно (<=): возвращает `True`, если левый операнд меньше или равен правому операнду.
    

Логические операторы используются для объединения логических значений или выражений и выполнения логических операций. К ним относятся следующие операторы:

- & (and): возвращает `True`, если оба операнда возвращают `True`
    
- | (or): возвращает `True`, если хотя бы один операнд возвращает `True`
    
- ~ (not): возвращает значение, противоположное логическому значению операнда.
    

### Логическое (булевое) индексирование

Для фильтрации строк в DataFrame на основе логических выражений Pandas применяет логическое (булевое) индексирование (Boolean Indexing). Оно включает использование булевых массивов или булевых серий для выбора определенных строк, которые удовлетворяют определенным условиям.

#### Фильтрация строк

Так, мы можем фильтровать строки на основе некоторого условия, создавая логическое выражение, которое для каждой строки принимает значение `True` или `False`. Например, получим города-миллионеры, где население меньше 1,5 млн. чел.:

```python
filtered_df = [df['Population'] < 1500]
>>> print(filtered_df)
         City  Population Time zone
4      Казань        1330     GMT+3
5  Красноярск        1212     GMT+7
6   Волгоград        1012     GMT+3
```

В этом примере выражение `df["Population"] < 1500]` генерирует логическую последовательность значений `True` и `False`, которые указывают, удовлетворяет ли каждая строка условию. Использование этой логической последовательности в качестве индекса в DataFrame (`df[значения True|False]`) возвращает только те строки, где соответствующие значения равны `True`.

Причем, выполнив собственно фильтрацию по условию, мы можем добавить фильтрацию по столбцам, выбрав только те столбцы, которые нужны:

```python
filtered_df = [df['Population'] < 1500][['City', 'Time zone']]
```

В данном случа еобратите внимание, что столбцы для выборки помещаются в квадратные скобки - `[["City", "Time zone"]]`, а не `["City", "Time zone"]`

Комбинируя логические выражения с помощью логических операторов, таких как & (and) и | (or), мы можем отфильтровать строки на основе нескольких условий. Например:

```python
filtered_df = [(df['Population'] < 1500) & (df['Time zone'] == 'GMT+3')]
>>>print(filtered_df)
         City  Population Time zone
4     Казань        1330     GMT+3
6  Волгоград        1012     GMT+3
```

Используя другой логический оператор - | (or), можно получить данные, которые отвечают хотя бы одному из условий:

```python
filtered_df = [(df['Population'] < 1500) | (df['Time zone'] == 'GMT+3')]
>>>print(filtered_df)
0           Москва       13274     GMT+3
1  Санкт-Петербург        5653     GMT+3
4           Казань        1330     GMT+3
5       Красноярск        1212     GMT+7
6        Волгоград        1012     GMT+3
```


И третий логический оператор - ~ (not) - унарный оператор, который возвращает `True`, если его операнд равен `False`, и наоборот. Например, получим строки, где в столбце "Time zone" значение НЕ "GMT+3":

```python
filtered_df =df[~df['Time zone'] == 'GMT+3']
           City  Population Time zone
2   Новосибирск        1637     GMT+7
3  Екатеринбург        1548     GMT+5
5    Красноярск        1212     GMT+7
```

То же самое мы могли бы получить с помощью оператора "!=":

```python
filtered_df = df[(df['Time zone'] != 'GMT+3')]
```

#### Фильтрация строк на основе подстрок

Отдельно стоит сказать про фильтрацию строковых значений - Pandas также позволяет фильтровать строки на основе совпадения подстрок. Для этого можно использовать методы строк и логические выражения. Например, давайте отфильтруем строки, где название города начинается с "К":

```python
filtered_df = df[df['City'].str.startswith('K')]
>>> print(filtered_df)
         City  Population Time zone
4      Казань        1330     GMT+3
5  Красноярск        1212     GMT+7
```

Здесь мы используем метод `str.startswith()` для создания булевой серии, которая указывает, начинается ли название каждого города с "К". Используя эту булеву серию для индексации, мы получаем строки, где условие истинно.

Приведу некоторые методы строк, которые мы можем применить в условиях:

- isalpha(): возвращает True, если строка состоит только из алфавитных символов
    
- islower(): возвращает True, если строка состоит только из символов в нижнем регистре
    
- isupper(): возвращает True, если все символы строки в верхнем регистре
    
- isdigit(): возвращает True, если все символы строки - цифры
    
- isnumeric(): возвращает True, если строка представляет собой число
    
- startswith(str): возвращает True, если строка начинается с подстроки str
    
- endswith(str): возвращает True, если строка заканчивается на подстроку str

# Методы фильтрации

Библиотека Pandas также предоставляет ряд специальных методов, которые позволяют получить некоторую порцию данных из DataFrame на основании определенных условий. Рассмотрим эти методы.

### query()

Метод query() в Pandas используется для фильтрации данных из DataFrame с помощью строкового выражения. Это выражение может включать имена столбцов из DataFrame, операторы сравнения (<, >, <=, >=, `==`, !=) и логические операторы (and, or, not). Метод `query()` предоставляет более читабельный и лаконичный способ фильтрации данных по сравнению с традиционным логическим индексированием.

Например, отфильтруем города, где население меньше 1500 тыс. чел. и одновременно часовой пояс GMT+3:

```python
filtered_df = df[(df['Population'] < 1500) & (df['Time zone'] == 'GMT+3')]
```

В данном случае использовалась логическое индексирование. С помощью метода `query()` та же самая фильтрация будет выглядеть следующим образом:

```python
filtered_df = df.query("Population < 1500 and `Time zone` == 'GMT+3'")
# следует обратить внимние, что для названиий столбцов с пробелами используется косые кавычки ``
# а для значения типа string следует обрамлять '' или "" для корректной работы метода
```

В обоих случаях результат будет один и тот же

Однако использование `query()` может улучшить читаемость кода, особенно при работе со сложными условиями, включающими несколько столбцов и операторов. И возможно, такой вариант придется больше по душе тем, кто привык или долгое время работал с запросами на SQL.

Тем не менее здесь надо сделать ряд примечаний. Я не случайно здесь использовал столбец "Time zone". Дело в том, что название этого столбца содержит пробел, поэтому мы не можем написать просто написать название, нам нужно название подобного столбца поместить в косые кавычки.

### between

Метод between() в Pandas используется для фильтрации данных из DataFrame на основе значений, которые попадают в указанный диапазон для данного столбца. Функция принимает два аргумента: нижнюю и верхнюю границы желаемого диапазона. Она применяет условие фильтрации к указанному столбцу и возвращает логический объект Series, указывающий, попадает ли значение каждой строки в этом столбце в указанный диапазон. Этот объект Series затем можно использовать для логической индексации с целью извлечения нужного подмножества данных:

```python
filtered_df = df[df['Population'].between(1212, 1637)]
>>> print(filtered_df)
           City  Population Time zone
2   Новосибирск        1637     GMT+7
3  Екатеринбург        1548     GMT+5
4        Казань        1330     GMT+3
5    Красноярск        1212     GMT+7
```

В этом примере мы используем метод `between()`, чтобы получить те города, где население в диапазоне от 1212 до 1637 тыс. чел.
### isin

Метод isin() используется для фильтрации данных из DataFrame на основе наличия значений столбца в заданном списке, кортеже или массиве. Метод `isin()` принимает один аргумент - список, кортеж или массив, который содержит значения , которые надо проверить. Этот метод применяет условие фильтрации к указанному столбцу и возвращает логический объект Series, указывающий, присутствует ли значение каждой строки в этом столбце в предоставленном списке. Например:

```python
filtered_df = df[df['City'].isin(['Новосибирск', 'Екатеринбург'])]
>>> print(filtered_df)
           City  Population Time zone
2   Новосибирск        1637     GMT+7
3  Екатеринбург        1548     GMT+5
```

Здесь выражение `df["City"].isin(["Новосибирск", "Екатеринбург"])` создает логический объект Series, который проверяет, присутствует ли каждое значение из столбца "Name" в предоставленном списке. Используя этот логический объект Series для индексации, мы получаем строки, где условие истинно.
### where()

Фильтрация по условию where() в Pandas включает создание маски на основе определенного условия и ее применение к DataFrame. При этом строки, которые удовлетворяют условию, сохраняются, а которые не соответствуют условию - заменяются на значение `NaN`. Этот метод сохраняет исходную форму DataFrame и обычно используется для условного маскирования и поэлементных операций:

```python
filtered_df = df.where(df['Population'] > 1548)
>>> print(filtered_df)
              City  Population Time zone
0           Москва     13274.0     GMT+3
1  Санкт-Петербург      5653.0     GMT+3
2      Новосибирск      1637.0     GMT+7
3              NaN         NaN       NaN
4              NaN         NaN       NaN
5              NaN         NaN       NaN
6              NaN         NaN       NaN
```

Здесь мы с помощью метода `where()` получаем все строки, где в столбце "Population" значение больше 1548. Несоответствующие условию строки заменяются значениями NaN

Если же надо удалить строки, где условие не выполняется, можно использовать метод `dropna()` после применения метода `where()`:

```python
filtered_df = df.where(df['Population'] > 1548).dropna()
```

# Фильтрация по дате и времени

Фильтрация по дате и времени в Pandas представляет использование различных методов для извлечения определенных подмножеств данных из DataFrame на основе атрибутов даты и времени. Эти методы особенно полезны при работе с временными рядами, позволяя сосредоточиться на определенных временных интервалах, датах или временных условиях для анализа.

### Индексирование по дате и времени

Индексирование по дате и времени в Pandas заключается в установке в качестве индекса DataFrame столбца, которые представляет значения даты и/или времени. Это позволяет легко и эффективно фильтровать и нарезать данные на основе атрибутов даты и времени. В этом случае можно напрямую указывать диапазоны дат для извлечения определенных подмножеств данных.

Для примера сначала рассмотрим, как создать временной ряд, который затем будет использоваться в качестве индекса:

```python
date_index = pd.date_range(start='2025-01-17', periods=5)
>>> print(date_index)
DatetimeIndex(['2025-01-17', '2025-01-18', '2025-01-19', '2025-01-20',
               '2025-01-21'],
              dtype='datetime64[ns]', freq='D')
```

Для создания временного ряда в библиотеке Pandas применяется метод `date_range()`, который генерирует индекс типа DatetimeIndex с фиксированной частотой (последовательность дат и времени). Ее ключевые параметры:

- `start`: левая граница (начальная дата) для генерации дат.
    
- `end`: правая граница (конечная дата) для генерации дат.
    
- `periods`: количество периодов (дат/временных точек) для генерации.
    
- `freq`: частота генерируемых дат, например, "D" (ежедневно), "H" (ежечасно), "M" (конец месяца), "B" (рабочий день) и т. д. По умолчанию - "D".
    

Здесь создается временной ряд, в котором временная метка первого элемента представляет дату "2025-01-17" (17 января). И всего генерируется пять подобных точек, которые соответствуют дням - с 17 по 21 января (так как для параметра `freq` значение по умолчанию - "D")

Далее мы можем использовать эти значения в качестве индекса для объекта DataFrame:

```python
# создаем временной ряд
date_index = pd.date_range(start="2025-01-17", periods=5)
# данные для DataFrame
temps = [-24, -27, -14, -10, -10]
# устанавливаем данные и индекс
df = pd.DataFrame({"temps": temps}, index=date_index)

>>> print(df)
filtered_df = df["2025-01-18":"2025-01-20"]
```

### `loc[]` с фильтрацией по дате и времени

Индексатор `loc[]` в Pandas представляет еще один способ выбора подмножеств данных из DataFrame с использованием меток. Индексатор loc[] можно комбинировать с логическими условиями для фильтрации данных на основе определенных диапазонов дат или времени:

```python
# создаем временной ряд
```

### Метод query() с фильтрацией по дате

Метод query() в Pandas также позволяет фильтровать DataFrame с помощью строкового выражения, которое включает условия даты и времени. Этот метод полезен для быстрого выбора подмножеств данных на основе определенных диапазонов дат:

# Фильтрация значений NULL или пропущенных значений

Пропущенные данные представляют отсутствие значений в определенных полях или столбцах набора данных. Это может происходить по различным причинам, таким как отсутствие сбора данных, повреждение данных во время хранения или передачи, или человеческие ошибки при вводе данных. На уровне кода подобные пропущенные значения обычно представлены либо константой `nan` из numpy или стандартным значением `None` в Python. Для обнаружения пропущенных данных в Pandas применяется ряд функций, таких как `isna()`, `isnull()` и `notnull()`.

### Функция isna()

Функция isna() используется для обнаружения пропущенных значений (`nan` или `None`) в DataFrame или Series. Она возвращает логическую маску той же формы, что и входной объект, где каждый элемент имеет значение `True`, если соответствующий элемент в DataFrame или Series отсутствует, и `False`, если элемент не отсутствует, что позволяет проверить наличия пропущенных данных. Например:

```python
import pandas as pd
import numpy as np

data = {
"A": [1, 2, np.nan, 4, None],
"B": [None, 6, 7, 8, np.nan]
}

df = pd.DataFrame(data)
# Использование isna() для обнаружения пропущенных значений в DataFrame
missing_data = df.isna()
>>> print(missing_data)
       A      B
0  False   True
1  False  False
2   True  False
3  False  False
4   True   True
```

Здесь столбцы "A" и "B" содержат пропущенные значения, представленные как `np.nan` и `None`. Метод isna() генерирует объект DataFrame той же формы, что и исходный DataFrame - df. Только в новом объекте DataFrame каждый элемент имеет значение `True`, если соответствующий элемент в df отсутствует (т.е. `np.nan` и `None`), или `False`, если элемент присутствует.

Например, в столбце "A" третья и пятая строки содержат пропущенные значения (True), так как они содержат `np.nan`/`None`, тогда как остальные строки содержат допустимые значения (False). В столбце "B" первая и пятая строки содержат пропущенные значения (True), так как они содержат `None` и `np.nan` соответственно, в то время как вторая, третья и четвертая строки содержат допустимые значения (False).

Используя данную функцию, мы можем создать маску, по которой затем отфильтровать данные в наборе. Например:

```python
data = {
"City": ["Москва", "Санкт-Петербург", "Новосибирск", "Екатеринбург", "Казань", "Красноярск", "Волгоград"],
"Population": [13274, None, 1637, None, 1330, 1212, None],
"Time zone": ["GMT+3", "GMT+3","GMT+7","GMT+5","GMT+3",None,"GMT+3"]
}

df = pd.DataFrame(data)
# Использование isna() для обнаружения пропущенных значений в DataFrame
missing_data = df[df["Population"].isna()]
>>> print(missing_data)
              City  Population Time zone
1  Санкт-Петербург         NaN     GMT+3
3     Екатеринбург         NaN     GMT+5
6        Волгоград         NaN     GMT+3
```

Здесь с помощью выражения `df["Population"].isna()` создаем маску для столбца "Population" - она возвращает объект Series, где каждый элемент равен `True`, если соответствующее значение в столбце "Population" отсутствует. Затем эта маска применяется для фильтрации строк, в которых отсутствует значение в столбце "Population".

Но что, если у нас отсутствующие значения есть в разных столбцах, и мы хотим отследить все строки с отсутствующими значениями вне зависимости от столбца? В этом случае мы можем применить к маске метод `any(axis=1)`. Например:

```python
missing_mask = df.isna()
missing_data = df[missing_mask.any(axis=1)]
>>> print(missing_data)
              City  Population Time zone
1  Санкт-Петербург         NaN     GMT+3
3     Екатеринбург         NaN     GMT+5
5       Красноярск      1212.0      None
6        Волгоград         NaN     GMT+3
```

Здесь метод `isna()` генерирует логическую маску `missing_mask`, где `True` также указывает на ячейки с отсутствующими значениями, а `False` - на ячейки с неотсутствующими значениями. Применив `any(axis=1)` к маске, мы определяем строки, в которых отсутствует значение в любом из столбцов. В итоге в результат также попадет строка, где также отсутствует значение в столбце "Time zone".
### Функция notna()

Метод notna() создает логическую маску, в которой значение каждой ячейки равно `True`, если значение не является пропущенным (не NaN), и `False` в противном случае. Эта маска может использоваться для фильтрации строк, в которых отсутствуют пропущенные значения. Например:

```python
data = {
"A": [1, 2, np.nan, 4, None],
"B": [None, 6, 7, 8, np.nan]
}

df = pd.DataFrame(data)
# Использование notna() для обнаружения НЕпропущенных значений в DataFrame
non_missing_mask = df.notna()
>>> print(non_missing_mask)
       A      B
0   True  False
1   True   True
2  False   True
3   True   True
4  False  False
```

Здесь мы видим, что только в строках с индексами 1 и 3 в обоих столбцах нет пропущенных значений, поэтому для обоих столбцов этих двух строк устанавливается значение `True`

Посмотрим на примере, как применить подобную маску, чтоы отфильтровать строки без пропущенных значений:

```python
data = {
"City": ["Москва", "Санкт-Петербург", "Новосибирск", "Екатеринбург", "Казань", "Красноярск", "Волгоград"],
"Population": [13274, None, 1637, None, 1330, 1212, None],
"Time zone": ["GMT+3", "GMT+3","GMT+7","GMT+5","GMT+3",None,"GMT+3"]
}

df = pd.DataFrame(data)
# маска для строк без пропущенных значений
non_missing_mask = df.notna()
non_missing_df = df[non_missing_mask.all(axis=1)]
>>> print(non_missing_df)
          City  Population Time zone
0       Москва     13274.0     GMT+3
2  Новосибирск      1637.0     GMT+7
4       Казань      1330.0     GMT+3
```

Здесь метод `notna()` генерирует логическую маску `non_missing_mask`, где `True` указывает на ячейки с непропущенными значениями, а `False` - на ячейки с пропущенными значениями. Применив функцию `all(axis=1)` к маске, мы точно определяем строки, в которых присутствуют значения во всех столбцах. Полученный DataFrame - `non_missing_df` содержит только строки, где нет пропущенных данных.
### Функция isnull()

Функция isnull() в Pandas используется для идентификации пропущенных значений (NaN или None) в DataFrame или Series. Эта функция генерирует логическую маску той же формы, что и исходный объект, где каждый элемент имеет значение `True`, если соответствующий элемент в DataFrame или Series отсутствует, или значение `False`, если элемент содержит допустимые данные. В целом функция `isnull()` эквивалентна использованию функции `isna()`, обе дают одинаковые результаты. Например, применим функцию `isnull()` для выявления пропущенных значений в DataFrame:

```python
data = {
"A": [1, 2, np.nan, 4, None],
"B": [None, 6, 7, 8, np.nan]
}

df = pd.DataFrame(data)
# Используем isnull() для поиска пропущенных значений в DataFrame
missing_data = df.isnull()
>>> print(missing_data)
       A      B
0  False   True
1  False  False
2   True  False
3  False  False
4   True   True
```

Применим метод isnull() для фильтрации данных - предположим, мы хотим отфильтровать строки, в которых столбец "Population" содержит нулевые значения. Мы можем сделать это с помощью метода isnull():

```python
data = {
"City": ["Москва", "Санкт-Петербург", "Новосибирск", "Екатеринбург", "Казань", "Красноярск", "Волгоград"],
"Population": [13274, None, 1637, None, 1330, 1212, None],
"Time zone": ["GMT+3", "GMT+3","GMT+7","GMT+5","GMT+3","GMT+7","GMT+3"]
}

df = pd.DataFrame(data)
filtered_df = df[df["Population"].isnull()]
>>> print(filtered_df)
              City  Population Time zone
1  Санкт-Петербург         NaN     GMT+3
3     Екатеринбург         NaN     GMT+5
6        Волгоград         NaN     GMT+3
```

Здесь выражение `df["Population"].isnull()` создает логический объект Series, который принимает значение True для строк, в которых столбец "Population" содержит нулевые значения. С точки зрения кода на Python это значения None. Если данные загружаются извне, то в этом случае данные могут быть просто не указаны, отсутствовать. В примере выше это характерно для трех городов.
### Функция notnull()

Функция notnull() используется для определения наличия непропущенных значений в DataFrame или Series. Она возвращает логическую маску той же формы, что и входной объект, где каждый элемент равен `True`, если соответствующий элемент в DataFrame или Series не отсутствует, и `False`, если элемент отсутствует (`numpy.nan`/`None`). Например:

```python
data = {
"A": [1, 2, np.nan, 4, None],
"B": [None, 6, 7, 8, np.nan]
}

df = pd.DataFrame(data)
# Используем notnull() для проверки значений на None в DataFrame
present_data = df.notnull()
>>> print(present_data)
       A      B
0   True  False
1   True   True
2  False   True
3   True   True
4  False  False
```

Фактически мы получаем результат, противоположный тому, что мы видели в случае с `isnull()`/`isna()`.

Опять же мы можем по этому методы создать маску для фильтрации строк, которые имеют данные в определенных столбцах:

```python
data = {
"City": ["Москва", "Санкт-Петербург", "Новосибирск", "Екатеринбург", "Казань", "Красноярск", "Волгоград"],
"Population": [13274, None, 1637, None, 1330, 1212, None],
"Time zone": ["GMT+3", "GMT+3","GMT+7","GMT+5","GMT+3",None,"GMT+3"]
}

df = pd.DataFrame(data)
# маска для тех строки, в которых есть значение в столбце Population
present_mask = df["Population"].notnull()
present_data = df[present_mask]
>>> print(present_data)
          City  Population Time zone
0       Москва     13274.0     GMT+3
2  Новосибирск      1637.0     GMT+7
4       Казань      1330.0     GMT+3
5   Красноярск      1212.0      None
```

Здесь получаем все строки, в которых есть значение в столбце Population.
