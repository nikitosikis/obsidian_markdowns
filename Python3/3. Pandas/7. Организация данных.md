
# Мультииндексирование в Series и DataFrame

Мультииндексирование (MultiIndexing), которая также известно как иерархическое индексирование (hierarchical indexing) представляет возможность установки несколько уровней индексирования для Series или DataFrame, что особенно полезно при работе с данными с несколькими измерениями или иерархией. Структура мультииндексирования позволяет организовывать и работать со сложными данными более структурированным и эффективным способом.

### Актуальность мультииндексов

Мы можем создать мультииндекс, используя список кортежей, где каждый кортеж представляет собой уникальную комбинацию значений индекса на каждом уровне иерархии. Каждый уровень в мультииндексе соответствует различному измерению или категории данных. Рассмотрим статистику с продажами трех товаров (ноутбуки, смартфоны, планшеты) за четыре разных года (с 2020 по 2023 год). Данные о продажах записаны в четырех разных валютах - долларах, евро, рублях и юанях, которые представлены соответственно метками USD, EUR, RUR, CNY:

```python
# Данные
sales_data = [100, 90, 105, 115, 85, 88, 95, 80, 110, 120, 100, 105]
 
# индексы в виде кортежей
index_tuples = [
    ("Laptop", 2020, "USD"),
    ("Smartphone", 2020, "EUR"),
    ("Tablet", 2020, "RUR"),
    ("Laptop", 2021, "USD"),
    ("Smartphone", 2021, "EUR"),
    ("Tablet", 2021, "RUR"),
    ("Laptop", 2022, "USD"),
    ("Smartphone", 2022, "EUR"),
    ("Tablet", 2022, "CNY"),
    ("Laptop", 2023, "USD"),
    ("Smartphone", 2023, "CNY"),
    ("Tablet", 2023, "RUR")
]
 
# Создание серии с мультииндексом
series = pd.Series(sales_data, index=index_tuples)
>>> print(series)
(Laptop, 2020, USD)        100
(Smartphone, 2020, EUR)     90
(Tablet, 2020, RUR)        105
(Laptop, 2021, USD)        115
(Smartphone, 2021, EUR)     85
(Tablet, 2021, RUR)         88
(Laptop, 2022, USD)         95
(Smartphone, 2022, EUR)     80
(Tablet, 2022, CNY)        110
(Laptop, 2023, USD)        120
(Smartphone, 2023, CNY)    100
(Tablet, 2023, RUR)        105
dtype: int64
```

Здесь список `index_tuples` определяет набор индексов, каждый из которых представляет собой кортеж c 3 элементами: тип товара, год и валюту продажи. Далее создаем серию, где в качестве данных применяется список с продажами - `sales_data`, а список `index_tuples` выполняет роль составного индекса.

Однако такая составная структура кортежа создает проблемы. Так, выполнение операций, основанных только на одном из этих измерений, может быть затруднительным. Например, если надо получить первую строку, то потребуется просмотреть весь кортеж:

```python
row_data = series[('laptop', 2020, 'USD')]
>>> print(row_data)
100
```

Однако в этом случае для получения данных необходимо знать и использовать весь кортеж. Если у нас только частичная информация (например, мы знаете только название товара и год, но не знаем валюту), то будет сложно получить доступ к нужным данным напрямую. А если мы захотим отфильтровать данные на основе определенного элемента кортежа (например, все продажи за 2021 год), то придется написать собственную логику фильтрации, что замедлит процесс и сделает его менее интуитивно понятным.

Кроме того, если мы попытаемся получить данные с помощью кортежа, которого нет в индексе (например, `("Laptop", 2020, "CNY")`), то это приведет к ошибке типа KeyError. Это делает процесс подверженным ошибкам и требует дополнительного кода для обработки исключений.

В итоге подобное использование составных индексов не очень эффективно с точки зрения вычислений, а также делает код менее читаемым и более сложным в сопровождении.

Для решения этой проблемы Pandas позволяяет применять Иерархическое индексирование или мультииндексирование (Hierarchical indexing / multi-indexing). Мультииндексирование позволяет индексировать данные на нескольких уровнях, тем самым более эффективно структурируя их. Вместо одного сложного индекса (кортежа из трех значений) мы можем использовать многоуровневый индекс, где каждый уровень соответствует одному из наших измерений (Товар, Год, Валюта). Кроме того, используя иерархическое индексирование, мы можем быстро и легко фильтровать и получать доступ к данным на основе любого уровня индекса, не утруждая себя запоминанием и разбором кортежей.

### Создание мультииндекса

Для создания мультииндекса библиотека Pandas предоставляет ряд методов: pandas.MultiIndex.from_tuples() и pandas.MultiIndex.from_product()

#### MultiIndex.from_tuples

Метод MultiIndex.from_tuples используется для создания многоуровневого индекса из списка кортежей. Каждый кортеж обычно представляет собой одну точку данных мультииндекса:

```python
MultiIndex.from_tuples(список_кортежей, names=[список_столбцов])
```

Первым параметром передается список кортежей, который будет служить в качестве индекса, в параметр `names` предоставляет метки для различных уровней иерархии (по сути для отдельных элементов в каждом кортеже из списка). Например:

```python
# Данные
sales_data = [100, 90, 105, 115, 85, 88, 95, 80, 110, 120, 100, 105]
 
# индексы в виде кортежей
index_tuples = [
    ("Laptop", 2020, "USD"),
    ("Smartphone", 2020, "EUR"),
    ("Tablet", 2020, "RUR"),
    ("Laptop", 2021, "USD"),
    ("Smartphone", 2021, "EUR"),
    ("Tablet", 2021, "RUR"),
    ("Laptop", 2022, "USD"),
    ("Smartphone", 2022, "EUR"),
    ("Tablet", 2022, "CNY"),
    ("Laptop", 2023, "USD"),
    ("Smartphone", 2023, "CNY"),
    ("Tablet", 2023, "RUR")
]
 
# Создание мультииндекса на основе списка кортежей
multiindex = pd.MultiIndex.from_tuples(index_tuples, names=["Product", "Year", "Currency"])
# Создание серии с мультииндексом
series = pd.Series(sales_data, index=multiindex)
>>> print(series)
Product      Year  Currency
Laptop      2020  USD         100
Smartphone  2020  EUR          90
Tablet      2020  RUR         105
Laptop      2021  USD         115
Smartphone  2021  EUR          85
Tablet      2021  RUR          88
Laptop      2022  USD          95
Smartphone  2022  EUR          80
Tablet      2022  CNY         110
Laptop      2023  USD         120
Smartphone  2023  CNY         100
Tablet      2023  RUR         105
dtype: int64
```

Здесь для создания индекса используется тот же список кортежей - `index_tuples`, который передается в качестве первого параметра в `MultiIndex.from_tuples`:

```python
multiindex = pd.MultiIndex.from_tuples(index_tuples, names=["Product", "Year", "Currency"])
```

Затем мы можем применить этот мультииндекс к нашим сериям:

```python
series = pd.Series(sales_data, index=multiindex)
```

Итак, в нашем случае результирующий объект мультииндекса представляет собой иерархический индекс с тремя уровнями: Product, Year и Currency. Это позволяет выполнять более сложные операции индексирования и сегментации. Например, допустим, нам надо получить все данные о продажах за 2021 год:

```python
data_2021 = series[:, 2021, :]
>>> print(data_2021)
Product      Currency
Laptop      USD         115
Smartphone  EUR          85
Tablet      RUR          88
dtype: int64
```

Для получения данных за 2021 год применяем выражение `series[:, 2021, :]`, где для первого и третьего столбцов ("Product" и "Currency") указывается оператор среза `:`, что эквивалентно получению всех данных по этому столбцу. А для второго столбца - "Year" указывается собственно критерий фильтрации - 2021.

Если нам надо найти, например, продажи по ноутбукам за 2021 год, то мы добавляем еще один критерий:

```python
laptops_2021 = series['Laptop':, 2021, :]
```

Если надо найти по первому столбцу, то можно указать только один критерий:

```python
laptops = series['Laptop']
>>> print(laptops)
Year  Currency
2020  USD         100
2021  USD         115
2022  USD          95
2023  USD         120
dtype: int64
```

### MultiIndex.from_product

Еще одна функция для создания мультииндекса - MultiIndex.from_product() позволяет создать мультииндекс, который представляет все возможные комбинации категорий данных. По сути этот метод создает мультииндекс из декартова произведения нескольких объектов. И вместо того, чтобы вручную перечислять все комбинации в виде кортежей (как в методе `MultiIndex.from_tuples()`), MultiIndex.from_product() автоматизирует этот процесс, генерируя все возможные комбинации предоставленных категорий.

Например, у нас есть три отдельных списка: один для товаров, один для лет и один для валют. Нам нужен мультииндекс, который представляет все комбинации этих списков:

```python
data = [120, 90, 105, 115]
 
# Разделение списков
products = ["Laptops", "Smartphones"]
years = [2024, 2025]
currencies = ["RUR", "USD"]
 
# Создание мультииндекса с помощью MultiIndex.from_product
multiindex = pd.MultiIndex.from_product([products, years, currencies], names=["Product", "Year", "Currency"])
 
# Настройка data под размер индекса
sales = pd.Series(data * 2, index=multiindex)
 
>>> print(sales)
Product      Year  Currency
Laptops      2024  RUR         120
                   USD          90
             2025  RUR         105
                   USD         115
Smartphones  2024  RUR         120
                   USD          90
             2025  RUR         105
                   USD         115
dtype: int64
```

Здесь метод `MultiIndex.from_product` создает мультииндекс из декартова произведения трех списков: products, years и currencies. Полученная серия имеет индекс, который представляет каждую комбинацию продукта, года и валюты.

Используя названия категорий и иерархический порядок индексов, мы можем обращаться к данным в серии. Например, получим данные о продажах ноутбуков за все годы и во всех валютах:

```python
data = [120, 90, 105, 115]
 
# списки с категориями
products = ["Laptops", "Smartphones"]
years = [2024, 2025]
currencies = ["RUR", "USD"]
 
# Создание мультииндекса
multiindex = pd.MultiIndex.from_product([products, years, currencies], names=["Product", "Year", "Currency"])
 
# Настройка data под размер индекса
sales = pd.Series(data * 2, index=multiindex)
 
# получаем данне по продажам в категории Laptops
laptops = sales["Laptops"]
>>> print(laptops)
Year  Currency
2024  RUR         120
      USD          90
2025  RUR         105
      USD         115
dtype: int64
```

### Мультииндексирование в DataFrame

В примерах выше мальтииндекс применялся к сериям, но аналогично мы можем работать и с наборами данных DataFrame:

```python
data = [
    [120, 105, 100, 101],
    [95, 91, 92, 93],
    [115, 116, 107, 109],
    [125, 106, 115, 108]
]
 
# списки с категориями
products = ["Laptops", "Smartphones"]
years = [2024, 2025]
currencies = ["RUR", "USD", "EUR", "CNY"]
 
# Создание мультииндекса
multiindex = pd.MultiIndex.from_product([products, years], names=["Product", "Year"])
 
# Настройка data под размер индекса
df = pd.DataFrame(data, index=multiindex, columns=currencies)
 
>>> print(df)
                  RUR  USD  EUR  CNY
Product     Year                    
Laptops     2024  120  105  100  101
            2025   95   91   92   93
Smartphones 2024  115  116  107  109
            2025  125  106  115  108
```

В отличие от Series, который имеет только один столбец данных и индекс, DataFrame позволяет использовать несколько столбцов данных, каждый из которых потенциально может иметь свой собственный тип данных. В данном примере в качестве столбцов используются валюты - значения из списка currencies. И как и в случае с сериями, нам надо смотреть, чтобы по количеству было соответствие. Так, в данных - списке data, у нас 16 элементов. Поэтому здесь используется 4 столбца - 4 валюты, а мультииндекс содержит 4 категории - декартово произведение 2 товаров на 2 года. В итоге мультииндекс создает 4 строки, которые при умножении на 4 столбца содают 16 ячеек для всех данных из списка data.

### Доступ к элементам в DataFrame с мультииндексом

Доступ к данным в DataFrame с несколькими индексами немного отличается от доступа к данным в обычном DataFrame из-за иерархической структуры индексов. Для доступа к отдельным столбцам, как и в общем случае, достаточно передать объекту DataFrame имя столбца. Например, выберем данные по столбцу "USD":

```python
sales_in_usd = df['USD']
>>> print(sales_in_usd)
Product      Year
Laptops      2024    105
             2025     91
Smartphones  2024    116
             2025    106
Name: USD, dtype: int64
```

Для доступа к строкам на основе иерархических индексов можно использовать индексатор `loc[]`. Например, получим из DataFrame данные о продажах ноутбуков за все годы и во всех валютах:

```python
laptops = df.loc['Laptops']
>>> print(laptops)
      RUR  USD  EUR  CNY
Year                    
2024  120  105  100  101
2025   95   91   92   93
```

Или получим данные о продажах ноутбуков только в 2024 году:

```python
laptops = df.loc[('Laptops', 2024)]
>>> print(laptops)
RUR    120
USD    105
EUR    100
CNY    101
Name: (Laptops, 2024), dtype: int64
```

Добавив название столбцов, мы можем получить данные непосредственно по этим столбцам. Например, получим данные о продажах ноутбуков только в 2024 году и в только в рублях:

```python
laptops = df.loc[('Laptops', 2024), 'RUR']
>>> print(laptops)
120
```

Также можно применять синтаксис срезов. Например, получим данные о ноутбуках в валютах, начиная USD до EUR:

```python
laptops = df.loc[('Laptops', 2024), 'USD':'RUR']
>>> print(laptops)
      USD  EUR
Year          
2024  105  100
2025   91   92
```

# Преобразование между форматами данных

## Длинный и широкий форматы

Данные бывают разных форм и размеров. Два распространенных формата - длинный (long format) и широкий (wide format), каждый из которых имеет свои преимущества и области применения. Библиотека Pandas предоставляет инструментарий для преобразования между этими форматами. Но, прежде чем перейти непосредственно к преобразованию, рассмотрим, что представляют эти форматы.

### Длинный формат

Длинный формат (long format), который также называют вертикальным форматом, предназначен для облегчения анализа и обработки. В этом формате каждая строка обычно представляет собой уникальное наблюдение, а каждый столбец — отдельную переменную или измерение. Некоторые ключевые характеристики длинного формата:

- Несколько строк на сущность: данные часто содержат несколько строк для каждой сущности. Эти строки содержат наблюдения, которые относятся к сущности.
    
- Названия переменных в одном столбце: вместо отдельных столбцов для каждой переменной, названия переменных обычно хранятся в одном столбце.
    
- Значения соответствуют наблюдениям: фактические значения переменных хранятся в отдельном столбце
    

Длинный формат особенно полезен для различных задач анализа данных, таких как агрегирование, фильтрация и группировка, хорошо подходит для данных временных рядов с несколькими временными точками для каждого объекта.

Пример данных в длинном формате:

```python
long_data = {
    "Company": ["Microsoft", "Microsoft", "Google", "Google", "Yandex", "Yandex"], 
    "Position": ["Developer", "Manager", "Developer", "Manager", "Developer", "Manager"], 
    "Salary": [1637, 1028, 1212, 1548, 1166, 1330]
}
 
long_df = pd.DataFrame(long_data)
>>> print(long_df)
     Company   Position  Salary
0  Microsoft  Developer    1637
1  Microsoft    Manager    1028
2     Google  Developer    1212
3     Google    Manager    1548
4     Yandex  Developer    1166
5     Yandex    Manager    1330
```

### Широкий формат

Широкий формат (wide format) характеризуется наличием отдельного столбца для каждой переменной или измерения. В этом формате каждая строка обычно представляет собой отдельную сущность или случай. Ключевые особенности широкого формата:

- Одна строка на сущность: каждая сущность имеет одну строку, что упрощает сравнение переменных между сущностями.
    
- Столбцы представляют переменные: переменные хранятся в виде столбцов, и заголовок каждого столбца указывает имя или категорию переменной.
    
- Значения соответствуют переменным: значения в каждой ячейке соответствуют наблюдениям для конкретной переменной.
    

Пример данных в широком формате:

```python
wide_data = {
    "Company": ["Microsoft", "Google", "Yandex"], 
    "Developer": [1637, 1212, 1166], 
    "Manager": [1028, 1548, 1330]
}
 
wide_df = pd.DataFrame(wide_data)
>>> print(wide_df)
     Company  Developer  Manager
0  Microsoft       1637     1028
1     Google       1212     1548
2     Yandex       1166     1330
```

Теперь рассмотрим, как выполнить преобразование из одного формата в другой.

## Стекирование и разблокировка

Изменение формы данных представляет одну из частых задач по управлению данными. Для изменения формы данных Pandas предоставляет ряд инструментов, в частности, функции stack и unstack. Эти функции позволяют преобразовывать данные, упрощая выполнение различных аналитических задач.

### Преобразование данных в вертикальный формат (Stacking)

Стекирование (stacking) (также называется "вертикальным преобразованием" или "vertical transformation"), представляет процесс перемещения столбцов в строки (из широкого формата в длинный формат). В результате чего получается вертикально расширенный набор данных. Стекирование полезно для таких задач, как агрегирование и фильтрация данных.

Рассмотрим следующий пример. Допустим, у нас есть набор данных с данными о зарплатах по двум должностям (Developer и Manager) в трех компаниях:

```
     Company  Developer  Manager
0  Microsoft       1637     1028
1     Google       1212     1548
2     Yandex       1166     1330
```

После преобразования в длинный, вертикальный формат структура данных будет выглядеть следующим образом:

```
     Company   Position  Salary
0  Microsoft  Developer    1637
1  Microsoft    Manager    1028
2     Google  Developer    1212
3     Google    Manager    1548
4     Yandex  Developer    1166
5     Yandex    Manager    1330
```

Для преобразования в длинный, вертикальный формат Pandas предоставляет функцию stack(). Рассмотрим ее применение:

```python
data = {
    "Company": ["Microsoft", "Google", "Yandex"], 
    "Developer": [1637, 1212, 1166], 
    "Manager": [1028, 1548, 1330]
}
 
df = pd.DataFrame(data)
stacked_df = df.set_index("Company").stack().reset_index(name="Salary")
   
>>> print(df)      
>>> print(stacked_df)
     Company  Developer  Manager
0  Microsoft       1637     1028
1     Google       1212     1548
2     Yandex       1166     1330

     Company    level_1  Salary
0  Microsoft  Developer    1637
1  Microsoft    Manager    1028
2     Google  Developer    1212
3     Google    Manager    1548
4     Yandex  Developer    1166
5     Yandex    Manager    1330
```

Итак, начальный объект DataFrame - df определен в широком формате и содержащий данные о зарплатах по должностям в трех компаниях. Каждая категория должностей (Developer, Manager) имеет свой собственный столбец, а данные о зарплатах разделены по компаниям (Microsoft, Google, Yandex).

При применении функции stack() с параметром `set_index` получается вертикально преобразованный DataFrame, часто называемый "стековым" DataFrame.

```python
stacked_df = df.set_index("Company").stack().reset_index(name="Salary")
```

Рассмотрим по шагово:

- `set_index("Company")`
    
    Столбец "Company" сохраняется в качестве переменной-идентификатора, а DataFrame индексируется по столбцу "Company"
    
- `stack()`
    
    Функция `stack` преобразует столбцы (Developer, Manager) в строки, фактически создавая многоуровневый индекс, где "Company" и безымянный столбец с должностями Developer/Manager становятся уровнями.
    
- `reset_index(name="Salary")`
    
    Функция `reset_index` используется для сброса индекса и создания нового столбца с именем "Salary" для хранения данных о зарплатах.
    

В результате получается DataFrame в длинном формате, который является более подходящим форматом для различных типов анализа и визуализации. Он упрощает манипулирование данными и обеспечивает структурированное представление данных о зарплатах по категориям компаний и должностей.

### Преобразование данных в горизонтальный формат (Unstacking)

Преобразование данных в горизонтальный формат ("горизонтальное преобразование") преобразует данные длинного формата в данные широкого формата. Оно включает в себя перемещение строк в столбцы, что приводит к горизонтально расширенному набору данных. Преобразование данных в горизонтальном формате часто используется для наглядной визуализации данных или для подготовки данных к моделированию.

Пусть у нас есть следующие данные:

```
     Company   Position  Salary
0  Microsoft  Developer    1637
1  Microsoft    Manager    1028
2     Google  Developer    1212
3     Google    Manager    1548
4     Yandex  Developer    1166
5     Yandex    Manager    1330
```

После горизонтального преобразования они будут выглядеть следующим образом:

```
Position    Company  Developer  Manager
0            Google       1212     1548
1         Microsoft       1637     1028
2            Yandex       1166     1330
```

Для подобного преобразования в Pandas применяется функция unstack():

```python
data = {
    "Company": ["Microsoft", "Microsoft", "Google", "Google", "Yandex", "Yandex"], 
    "Position": ["Developer", "Manager", "Developer", "Manager", "Developer", "Manager"], 
    "Salary": [1637, 1028, 1212, 1548, 1166, 1330]
}
 
df = pd.DataFrame(data)
unstacked_df = df.set_index(["Company", "Position"])["Salary"].unstack().reset_index()

>>> print(df)      
>>> print(unstacked_df)
     Company   Position  Salary
0  Microsoft  Developer    1637
1  Microsoft    Manager    1028
2     Google  Developer    1212
3     Google    Manager    1548
4     Yandex  Developer    1166
5     Yandex    Manager    1330

Position    Company  Developer  Manager
0            Google       1212     1548
1         Microsoft       1637     1028
2            Yandex       1166     1330
```

Здесь изначально набор данных DataFrame - df определен в длинном формате, который содержит данные о зарплатах для двух должностей в разных компаниях. Для преобразования этого набора данных в широкий формат применяем вызов:

```python
unstacked_df = df.set_index(["Company", "Position"])["Salary"].unstack().reset_index()
```

Результатом является разбитый DataFrame, где для каждой должности в столбце "Position" создается свой собственный столбец, который отображает зарплату ("Salary") для каждой компании. Такой формат упрощает сравнение данных между компаниями и должностями.

# Преобразование данных в строки (Melting)

Melting (сплавление, соединение) представляет преобразование данных из широкого формата в длинный формат путем изменения их формы. Оно похоже на стекирование (stacking) - преобразование столбцов в строки, в результате которого получается вертикально расширенный набор данных.

Для данного типа преобразований в Pandas применяется функция melt():

```python
pandas.melt(frame, id_vars=None, value_vars=None, var_name=None, value_name="value", col_level=None, ignore_index=True)
```

Рассмотрим ключевые параметры функции:

- `id_vars`: указывает, какие столбцы должны оставаться неизмененными (непреобразованными в длинный формат) и не будут преобразовываны в строки.
    
- `value_vars`: указывает, какие столбцы должны быть преобразованы в строки в длинном формате. Если не указано, будут преобразованы все столбцы, не упомянутые в `id_vars`.
    
- `var_name`: позволяет указать имя столбца, который будет хранить имена переменных после преобразования в длинный формат.
    
- `value_name`: используется для указания имени столбца, который будет хранить значения после преобразования в длинный формат.
    

Рассмотрим следующий пример. Предположим, у вас есть данные временных рядов в широком формате:

|            |                   |                  |
| ---------- | ----------------- | ---------------- |
| Дата       | Макс. температура | Мин. температура |
| 2026-01-21 | -8                | -11              |
| 2026-01-22 | -11               | -17              |
| 2026-01-23 | -14               | -22              |

Здесь данные представлены в широком формате, который отображает данные временных рядов - даты, значения минимальных и максимальных дневных температур. Посмотрим, как преобразовать эти данные с помощью функции melt():

```python
data = {
    "Date": ["2026-01-21", "2026-01-22", "2026-01-23"],
    "Max Temp": [-8, -11, -14],
    "Min Temp": [-11, -17, -22]
}
 
df = pd.DataFrame(data)
 
# Преобразование данных с помощью функции melt
melted_df = df.melt(id_vars=["Date"], var_name="Variable", value_name="Value")
>>> print(df)
>>> print(melted_df)
         Date  Max Temp  Min Temp
0  2026-01-21        -8       -11
1  2026-01-22       -11       -17
2  2026-01-23       -14       -22

         Date  Variable  Value
0  2026-01-21  Max Temp     -8
1  2026-01-22  Max Temp    -11
2  2026-01-23  Max Temp    -14
3  2026-01-21  Min Temp    -11
4  2026-01-22  Min Temp    -17
5  2026-01-23  Min Temp    -22
```

Здесь при вызове функции `melt()` мы назначаем столбец "Date" в качестве переменной-идентификатора (`id_vars`), которая остается неизменной. Мы также устанавливаем параметр `var_name` в значение "Variable" (значение произвольное), а параметр `value_name` - в значение "Value" (значение также произвольное). В результате преобразования получается DataFrame в длинном формате - объект melted_df, который может быть более удобен для последующего анализа и визуализации.

# Развертывание данных

В анализе данных часто встречаются вложенные структуры или списки, которые необходимо развернуть для дальнейшего исследования. Развертывание данных (Exploding Data) представляет преобразование вложенных или спископодобных структур внутри DataFrame в отдельные строки. Эта операция расширяет данные, делая их более доступными для анализа.

Для развертывания данных библиотека Pandas предоставляет функцию explode():

```python
DataFrame.explode(column, ignore_index=False)
```

Ее ключевой параметр - `column` указывает, в каком столбце находится вложенная структура, которую надо развернуть.

Предположим, у нас есть набор данных о программистах и используемых ими языках программирования. Каждый программист может использовать несколько языков программирования, и эта информация хранится в списке:

| Имя | Возраст | Языки                |
| --- | ------- | -------------------- |
| Tom | 41      | [Python, JavaScript] |
| Bob | 56      | [Java, C#]           |
| Sam | 32      | [Python]             |

Для более эффективного анализа этих данных мы можем использовать функцию `explode`, которая позволит развернуть столбец "Языки", создавая отдельные строки для каждого языка для каждого программиста:

```python
data = {
    "Name": ["Tom", "Bob", "Sam"],
    "Age": [41, 56, 32],
    "Langs": [["Python", "JavaScript"], ["Java", "C#"], ["Python"]]
}
 
df = pd.DataFrame(data)
# Преобразование данных с помощью функции explode
exploded_df = df.explode("Langs", ignore_index=True)
>>> print(df)
>>> print(exploded_df)
  Name  Age                 Langs
0  Tom   41  [Python, JavaScript]
1  Bob   56            [Java, C#]
2  Sam   32              [Python]

  Name  Age       Langs
0  Tom   41      Python
1  Tom   41  JavaScript
2  Bob   56        Java
3  Bob   56          C#
4  Sam   32      Python
```

Здесь в функцию `explode` передается название столбца "Langs", благодаря чему вложенные списки в столбце "Langs" разворачиваются в отдельные строки, в результате чего получаются отдельные строки для каждой пары программист-язык. А параметр `ignore_index=True` позволяет преобразовать эти данные в новый DataFrame с именем exploded_df.

# Создание сводных таблиц

Сводные таблицы представляют эффективный инструмент для преобразования данных, их реорганизации и создания сводки. Они особенно полезны для агрегирования данных, предоставляя сжатое представление больших наборов данных путем группировки и обобщения информации на основе определенных критериев. Сводные таблицы широко используются в анализе данных и отчетности для получения ценных выводов и принятия решений на основе данных.

Сводные таблицы имеют ряд характеристик:

- `Индекс`: определяет строки таблицы. Он представляет категориальные переменные, по которым надо сгруппировать и обобщить данные.
    
- `Столбцы`: представляют переменные, по которым надо сравнить или классифицировать данные.
    
- `Значения`: данные, которые надо обобщить или агрегировать с помощью таких функций агрегирования.
    
- `Функции агрегирования`: определяют, как данные обобщаются в сводной таблице
    

В Pandas для создания сводных таблиц применяется функция pivot_table():

```python
pandas.pivot_table(data, values=None, index=None, columns=None, aggfunc="mean", fill_value=None, margins=False, dropna=True, margins_name="All", observed=<no_default>, sort=True)
```

Ее основные параметры:

- `data`: объект DataFrame, для которого создается сводная таблица
    
- `values`: столбец или столбцы для агрегирования
    
- `index`: столбец или список столбцов для создания индекса
    
- `columns`: столбец или список столбцов для для группировки
    
- `aggfunc`: агрегатная функция или список функций (`count, sum, max` и т.д.)
    
- `fill_value`: значение для замены пропущенных значений (в результирующей сводной таблице после агрегирования).
    

Рассмотрим ее использование. Например, у нас есть набор данных о продажах различных товаров в ряд стран и выручкой от соответствующих продаж:

# Объединение данных по строкам и столбцам

Объединение данных (Concatenating) включает в себя комбинирование нескольких наборов данных для создания большего набора данных. Эта операция может выполняться либо по строкам (вертикально), либо по столбцам (горизонтально). Для объединения наборов данных Pandas предоставляет функцию pandas.concat():

```python
result = pandas.concat([dataframe1, dataframe2, ...], axis=0)
```

В качестве параметров функция принимает список наборов данных DataFrame, которые надо объединить. А параметр `axis` указывает ось, вдоль которой будет происходить объединение. Значение `axis=0` применяется для объединения по строкам (вертикальное объединение) и `axis=1` - для объединения по столбцам (горизонтальное объединение).
### Объединение данных по строкам

Объединение данных по строкам или вертикальное объединение представляет объединение нескольких наборов данных путем последовательного добавления их строк. Это полезно, когда есть наборы данных с одинаковыми столбцами, и надо увеличить количество строк в результирующем наборе данных. Рассмотрим пример, где у нас есть два набора данных, df1 и df2, которые содержат информацию о сотрудниках и которые надо объединить:

```python
df1 = pd.DataFrame({
    "Name": ["Tom", "Alice"],
    "Age": [41, 38],
    "Position": ["Developer", "Manager"]
})
 
df2 = pd.DataFrame({
    "Name": ["Bob", "Sam"],
    "Age": [56, 32],
    "Position": ["Developer", "Manager"]
})
 
# Объединение строк из наборов df1 и df2
all_df = pd.concat([df1, df2], axis=0)
>>> print(all_df)
    Name  Age   Position
0    Tom   41  Developer
1  Alice   38    Manager
0    Bob   56  Developer
1    Sam   32    Manager
```

Здесь полученный набор данных all_df содержит все строки из обоих датафреймов: df1 и df2. Столбцы остаются неизменными, а значения индексов сохраняются из исходных фреймов данных.

При этом отдельные наборы данных могут быть неполными, например, не иметь каких-то отдельных столбцов. В этом случае соответствующие значения заполняются значением `NaN`:

```python
df1 = pd.DataFrame({
    "Name": ["Tom", "Alice"],
    "Age": [41, None],
    "Position": ["Developer", "Manager"]
})
 
df2 = pd.DataFrame({
    "Name": ["Bob", "Sam"],
    "Age": [56, 32]
})
 
# Объединение строк из наборов df1 и df2
all_df = pd.concat([df1, df2], axis=0)
>>> print(all_df)
    Name   Age   Position
0    Tom  41.0  Developer
1  Alice   NaN    Manager
0    Bob  56.0        NaN
1    Sam  32.0        NaN
```
### Объединение данных по столбцам

Объединение данных по столбцам включает в себя комбинирование нескольких наборов данных путем сложения их столбцов. Эта операция полезна, когда есть наборы данных с различными значениями, и необходимо расширить информацию, добавив новые атрибуты (столбцы). Например, у нас есть два набора данных - df1 и df2 с информацией о сотрудниках. Эти наборы данных имеют один и тот же индекс (представляющий сотрудников), но содержат разные атрибуты:

```python
df1 = pd.DataFrame({
    "Name": ["Tom", "Bob", "Sam"],
    "Age": [41, 56, 32]
})
 
df2 = pd.DataFrame({
    "Position": ["Developer", "Developer", "Manager"],
    "Salary": [1200, 1300, 1100]
})
 
# Объединение столбцов из наборов df1 и df2
all_df = pd.concat([df1, df2], axis=1)
>>> print(all_df)
  Name  Age   Position  Salary
0  Tom   41  Developer    1200
1  Bob   56  Developer    1300
2  Sam   32    Manager    1100
```

Здесь набор данных df1 содержит имена и возраст сотрудников, а df2 - должности и зарплаты соответствующих сотрудников. Функция `pd.concat()` выравнивает столбцы из обоих датафреймов, создавая объединенные столбцы.

# Соединение наборов DataFrame

Слияние датафреймов (merging) представляет комбинирование наборов данных на основе общих столбцов, которые называются ключами. Эта операция аналогична объединению данных их баз данных, где строки из разных таблиц выравниваются на основе общих значений в указанных столбцах. Слияние позволяет консолидировать данные из нескольких источников в один датафрейм.

Для слияния датафреймов библиотека Pandas предоставляет функцию merge():

```python
result = pandas.merge(left_dataframe, right_dataframe, on="столбец_ключ", how="принцип_объединения")
```

Параметры функции:

- `left_dataframe`: набор данных DataFrame, который надо объединить с "левой стороны"
    
- `right_dataframe`: набор данных DataFrame, который надо объединить с "правой стороны"
    
- `on`: столбец(ы), используемые в качестве ключа для объединения
    
- `how`: тип соединения. Он может принимать следующие значения:
    
    - inner: внутренее соединение
        
    - left: левое соединение
        
    - right: правое соединение
        
    - outer: внешнее соединение
        

В зависимости от параметра `how` есть разные типы соединения, которые соответственно дают разный результат при соединении одних и тех же данных. Рассмотрим эти типы соединений.

### Inner Join

Inner Join или внутренние соединения - это тип операции слияния, который объединяет фреймы данных на основе общего ключа, сохраняя только те строки, где значения ключа присутствуют в обоих фреймах данных. Внутреннее соединение акцентирует внимание на общей информации и предоставляет консолидированное представление пересечений данных. То есть `inner join` фокусируется на пересечении значений ключа между фреймами данных, отбрасывая строки, которые не совпадают по значению. Внутренние соединения особенно полезны для извлечения информации об элементах, общих для нескольких наборов данных.

Например, у нас есть два набора данных, orders и clients, которые хранят соответственно информацию о заказах и о клиентах, сделавших эти заказы. И нам надо соединить эти данные в общий набор данных:

```python
clients = pd.DataFrame({
 
    "Client_ID": [1, 2, 3, 4],
    "Name": ["Tom", "Bob", "Sam", "Alice"],
    "City": ["Москва", "Санкт-Петербург", "Волгоград", "Воронеж"]
})
 
orders = pd.DataFrame({
 
    "Order_ID": [101, 102, 103],
    "Client_ID": [1, 2, 4],
    "Product": ["Смартфон", "Планшет", "Ноутбук"]
})
 
# Внутреннее соединение наборов данных orders и clients на основе столбца-ключа "Client_ID"
result = pd.merge(clients, orders, on="Client_ID", how="inner")
>>> print(result)
   Client_ID   Name             City  Order_ID   Product
0          1    Tom           Москва       101  Смартфон
1          2    Bob  Санкт-Петербург       102   Планшет
2          4  Alice          Воронеж       103   Ноутбук
```

Здесь мы выполняем внутреннее соединение датафреймов orders и clients по столбцу "Client_ID". В результате возвращаемый набор данных - result сохраняет только те строки, где значения "Client_ID" присутствуют в обоих датафреймах. Несовпадающие строки исключаются из вывода.

### Outer Join

Outer Join или внешнее соединение представляет операцию слияния, которая объединяет два датафрейма на основе указанного ключевого столбца, сохраняя все строки из обоих датафреймов и заполняя пустые ячейки значениями `NaN`. Внешние соединения подчеркивают объединение данных, позволяя получить информацию из обоих наборов данных, даже если ключевые значения не совпадают идеально.

Например, возьмем данные из предыдущего примера, то теперь применим внешнее объединение:

```python
clients = pd.DataFrame({
 
    "Client_ID": [1, 2, 3, 4],
    "Name": ["Tom", "Bob", "Sam", "Alice"],
    "City": ["Москва", "Санкт-Петербург", "Волгоград", "Воронеж"]
})
 
orders = pd.DataFrame({
 
    "Order_ID": [101, 102, 103],
    "Client_ID": [1, 2, 4],
    "Product": ["Смартфон", "Планшет", "Ноутбук"]
})
 
# Внешнее соединение наборов данных orders и clients на основе столбца-ключа "Client_ID"
result = pd.merge(clients, orders, on="Client_ID", how="outer")
>>> print(result)
   Client_ID   Name             City  Order_ID   Product
0          1    Tom           Москва     101.0  Смартфон
1          2    Bob  Санкт-Петербург     102.0   Планшет
2          3    Sam        Волгоград       NaN       NaN
3          4  Alice          Воронеж     103.0   Ноутбук
```

Здесь мы также выполняем соединение датафреймов orders и clients по столбцу "Client_ID", но теперь это внешнее соединение. В итоге функция `pd.merge()` соединяет все строки из обоих датафреймов, заполняя пропущенные значения `NaN` там, где это необходимо. Полученный датафрейм содержит всех клиентов, даже если у них нет заказов.

### Left Outer Join

Левое внешнее соединение или Left Outer Join представляет операцию слияния, которая объединяет два датафрейма на основе указанного ключевого столбца, сохраняя все строки из левого датафрейма. Если значения ключа совпадают, данные из правого датафрейма включаются; в противном случае пропущенные значения заполняются `NaN`. Левые внешние соединения направлены на сохранение данных из левого датафрейма, одновременно добавляя соответствующую информацию из правого датафрейма.

Для примера возьмем те же данные, что и в предыдущем коде, то теперь применим левое внешнее объединение:

```python
clients = pd.DataFrame({
 
    "Client_ID": [1, 2, 3, 4],
    "Name": ["Tom", "Bob", "Sam", "Alice"],
    "City": ["Москва", "Санкт-Петербург", "Волгоград", "Воронеж"]
})
 
orders = pd.DataFrame({
 
    "Order_ID": [101, 102, 103],
    "Client_ID": [1, 2, 4],
    "Product": ["Смартфон", "Планшет", "Ноутбук"]
})
 
# Левое внешнее соединение наборов данных orders и clients на основе столбца-ключа "Client_ID"
result = pd.merge(clients, orders, on="Client_ID", how="left")
>>> print(result)
   Client_ID   Name             City  Order_ID   Product
0          1    Tom           Москва     101.0  Смартфон
1          2    Bob  Санкт-Петербург     102.0   Планшет
2          3    Sam        Волгоград       NaN       NaN
3          4  Alice          Воронеж     103.0   Ноутбук
```

Если мы посмотрим на результат программы, то мы увидим, что он похож на результат из предыдущего примера с внешним соединением.

Но механика самого соединения отличается. Здесь когда значения "Client_ID" совпадают между датафреймами clients и orders (например, для клиентов Tom, Bob, Alice), соответствующие значения "Order_ID" и "Product" включаются в объединенный результат. Для клиента "Sam" с "Client_ID" 3, у которого нет соответствующей записи в датафрейме orders, столбцы "Order_ID" и "Product" заполняется значением `NaN`.

Но если мы изменим левую таблицу - поменяем местами clients и orders:

```python
result = pd.merge(orders, clients, on="Client_ID", how="left")
>>> print(result)
   Order_ID  Client_ID   Product   Name             City
0       101          1  Смартфон    Tom           Москва
1       102          2   Планшет    Bob  Санкт-Петербург
2       103          4   Ноутбук  Alice          Воронеж
```

То мы получим несколько другой результат, хотя данные остаются теми же и столбец-ключ тот же, но теперь заполнение идет по новому левому датафрейму - по набору orders.

### Right Join

Right Join или правое соединение представляет операцию слияния, которая объединяет два датафрейма на основе указанного ключевого столбца, сохраняя все строки из правого датафрейма. Если значения ключа совпадают, данные из левого датафрейма включаются; в противном случае пропущенные элементы заполняются значением `NaN`. Правое соединение направлено на сохранение данных из правого датафрейма при одновременной интеграции соответствующей информации из левого датафрейма.

Правые cоединения важны для сохранения данных из правого датафрейма при одновременном включении соответствующей информации из левого датафрейма. Это гарантирует, что никакая информация из правого датафрейма не будет потеряна в процессе слияния, обеспечивая сбалансированное представление взаимосвязи между наборами данных.

Для примера опять же возьмем те же данные из примеров выше, только теперь выполним правое соединение:

```python
clients = pd.DataFrame({
 
    "Client_ID": [1, 2, 3, 4],
    "Name": ["Tom", "Bob", "Sam", "Alice"],
    "City": ["Москва", "Санкт-Петербург", "Волгоград", "Воронеж"]
})
 
orders = pd.DataFrame({
 
    "Order_ID": [101, 102, 103],
    "Client_ID": [1, 2, 4],
    "Product": ["Смартфон", "Планшет", "Ноутбук"]
})
 
# Правое соединение наборов данных orders и clients на основе столбца-ключа "Client_ID"
result = pd.merge(orders, clients, on="Client_ID", how="right")
>>> print(result)
   Order_ID  Client_ID   Product   Name             City
0     101.0          1  Смартфон    Tom           Москва
1     102.0          2   Планшет    Bob  Санкт-Петербург
2       NaN          3       NaN    Sam        Волгоград
3     103.0          4   Ноутбук  Alice          Воронеж
```

Здесь все строки из датафрейма orders сохраняются в объединенном результате. Если значения "Client_ID" совпадают между датафреймами clients и orders (Client_ID = 1, 2, 4), соответствующая информация из набора clients включается в объединенный результат. Для клиента с идентификатором Client_ID=3, у которого нет заказов - соответствующей записи в датафрейме orders, столбцы "Order_ID" и "Product" заполняются значениями `NaN`.

### Соединение датафреймов по их индексу

При соединении датафреймов в качестве основы для их слияния также можно использовать их индексы. Это позволяет выравнивать данные на основе значений индексов датафреймов, а не использовать определенный столбец. Вот как это можно сделать с помощью Pandas. Для этого в функции `pd.merge()` дополнительно устанавливаются два параметра:

- `left_index`: принимает значение `True`, если для слияния надо использовать индекс левого датафрейма
    
- `right_index`: принимает значение `True`, если для слияния надо использовать индекс правого датафрейма
    

Рассмотрим два датафрейма, students и scores, где индексы представляют собой идентификаторы студентов, и надо выполнить соединение наборов данных на основе этих индексов:

```python
students = pd.DataFrame({
    "Name": ["Tom", "Bob", "Sam"],
    "Age": [21, 22, 20]
    }, index=[101, 102, 103])
 
scores = pd.DataFrame({
    "Python": [95, 85, 78],
    "Java": [89, 92, 81]
}, index=[101, 102, 104]) # Обратите внимание на отсутствующий индекс 103
 
# Внутреннее соединение на основе индексов
result = pd.merge(students, scores, left_index=True, right_index=True, how="inner")
>>> print(result)
    Name  Age  Python  Java
101  Tom   21      95    89
102  Bob   22      85    92
```

Здесь датафреймы students и scores представляют соответственно студентов и их оценки. Благодаря установке `left_index=True` и `right_index=True` в вызове функции `pd.merge()` указываем, что оба с датафрейма будут соединяться на основе их индексов. А, установив `how="inner"`, мы указываем, что хотим выполнить внутреннее соединение (inner join).

В результате генерируемый набор данных result будет содержать только строки с индексами, общими для students и scores, в данном случае это 101 и 102. Строки, соответствующие индексам 103 (присутствуют только у students) и 104 (присутствуют только у scores), исключаются, поскольку у них нет аналога в другом наборе данных.

### Объединение по нескольким столбцам

Соединение датафреймов по нескольким столбцам обеспечивает более точное выравнивание строк, что может быть полезно в сценариях, когда одного столбца недостаточно для определения уникального совпадения. в вызове функции `pd.merge()` через параметр `on` указывается список ключевых столбцов. Например, у вас есть два датафрейма - один с данными о студентах, а другой с их оценками. Оба датафрейма содержат столбцы "StudentID" и "Subject", и необходимо соединить их на основе этих столбцов:

```python
students = pd.DataFrame({
    "StudentID": [101, 101, 102, 102],
    "Subject": ["Python", "Java", "Python", "Java"],
    "Name": ["Tom", "Tom", "Bob", "Bob"]
})
 
scores = pd.DataFrame({
    "StudentID": [101, 101, 102, 102],
    "Subject": ["Python", "Java", "Python", "Java"],
    "Score": [80, 95, 68, 98]
})
 
# Внутреннее соединение по нескольким столбцам
result = pd.merge(students, scores, on=["StudentID", "Subject"], how="inner")
>>> print(result)
   StudentID Subject Name  Score
0        101  Python  Tom     80
1        101    Java  Tom     95
2        102  Python  Bob     68
3        102    Java  Bob     98
```

В качестве типа соединения здесь применяется inner join. Pltcm Здесь в вызове функции `merge()` применяется параметр `on=["StudentID", "Subject"]`, благодаря чему наборы данных students и scores соединяются на основе значений в столбцах "StudentID" и "Subject". Это означает, что строка в наборе данных students, например, со значениями `StudentID=1` и `Subject="Python"`, будет объединена со строкой в датафрейме scores, которая имеет те же значения столбцов StudentID и Subject. В результате датафрейм будет содержать строки со столбцами "StudentID", "Subject", "Name" и "Score".
