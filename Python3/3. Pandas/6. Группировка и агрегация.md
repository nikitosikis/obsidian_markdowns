
Группировка представляет процесс разделения данных на группы на основе одного или нескольких критериев. Это может быть категориальная переменная или комбинация переменных. Группировка помогает создавать подмножества данных, имеющие схожие характеристики. После группировки можно выполнять различные операции над каждой группой независимо.

### Группировка по принципу "разделить-применить-объединить"

Концепция группировки по принципу "разделить-применить-объединить" (Split-Apply-Combine) представляет фундаментальный метод, который позволяет аналитикам данных извлекать значимые выводы из наборов данных. Этот процесс включает три основных шага:

1. Разделение данных: на этом начальном этапе набор данных делится на отдельные группы на основе определенных атрибутов или критериев. Например, чтобы лучше понять показатели продаж по продуктам, мы можем разделить данные о продажа на группы на основе различных категорий продуктов.
    
2. Применение функции: после группировки данных следующий шаг включает применение функции к каждой группе. Эта функция может быть любой, которая помогает нам извлечь значимые выводы из данных. Например, мы можем захотеть рассчитать средние продажи для каждой категории продуктов и т.д.
    
3. Объединение результатов: после применения функции к каждой группе мы объединяем результаты. В нашем примере средние продажи, рассчитанные для каждой категории товаров, объединяются для формирования общего сводного отчета. Этот отчет показывает эффективность различных категорий товаров с точки зрения продаж.
    

Предположим, у нас есть следующие данные:

```
|Товар|Продажи|
|A|100|
|B|160|
|A|200|
|B|120|
```

Допустим, нам надо подсчитать среднее количество продаж по каждому товару. Общий процесс для подобных данных будет выглядеть следующим образом:

1. Разделение данных. Начнем с группировки данных по столбцу "Товар":

```
Группа товара A: [100, 200]
Группа товара B: [160, 120]
```

2. Применение функции. Применим функцию для нахождения среднего значения по каждой группе:

```
Средние продажи по группам:

Группа товара A: (100 + 200) / 2 = 150
Группа товара B: (160 + 120) / 2 = 140
```

3. Объединение результатов. Вычисленные средние продажи объединяются для получения сводки:

```
Группа товара A: Средние продажи = 150
Группа товара B: Средние продажи = 140
```

Посмотрим, как мы можем это автоматизировать с помощью функционала Pandas. Для разделения объекта на группы в Pandas у объекта DataFrame применяется метод groupby:

```python
grouped_date = dataframe.groupby('стобец_для_группировки')
```

В метод передается метка столбца, по которому надо сгруппировать данные. Результат метода - это результат операции группировки, который представляет сгруппированные данные. Например:

# Агрегация

Агрегация представляет процесс применения функции к каждой группе и объединения результатов в компактную форму для анализа. Функции агрегации могут включать сумму, среднее значение, количество, максимум, минимум и другие. Эти функции помогают получать значимые статистические данные и сводки из сгруппированных данных, выявляя тенденции, закономерности и взаимосвязи.

Pandas предоставляет ряд встроенных методов для агрегирования данных. Некоторые распространенные методы:

- mean(): вычисляет среднее значение столбца.
    
- sum(): cуммирует значения столбца, показывая общую сумму.
    
- min(): определяет наименьшее значение в столбце.
    
- max(): находит наибольшее значение в столбце.
    
- count(): вычисляет количество ненулевых значений в столбце.
    
- median(): вычисляет среднее значение в столбце.
    
- std(): измеряет разброс значений в столбце.
    
- var(): вычисляет изменчивость значений в столбце.
    

Все эти методы имеют одинаковый синтаксис:

# Агрегация с помощью функции agg()

С помощью функции agg() Pandas позволяет применить различные функций к столбцам DataFrame и рассчитать различные сводные статистические данные или агрегатные значения для определенных столбцов DataFrame. Этот полезно для быстрого извлечения из данных нескольких аналитических сведений за одну операцию. Например, можно вычислить среднее значение, сумму, максимум, минимум или другие статистические данные для определенных столбцов за один раз. Это особенно полезно, когда надо анализировать различные аспекты данных по нескольким столбцам одновременно.

Функция agg() имеет следующий синтаксис:

```python
result = df.agg({"Столбец1": "функция1", "Столбец2": "функция2", ...})
```

В функцию передается словарь, где каждый элемент представляет пару "Столбец - функция", то есть с именем столбца DataFrame сопоставляется функция агрегирования, которую надо применить к столбцу. Например:

```python
result = df.agg({'Population': 'min', 'City': 'count'})
>>> print(result)
Population    1012
City            16
dtype: int64
```

Здесь для столбца "Population" вычисляем минимальное значение (применяем функцию 
`min`), а для столбца "City" подсчитываем общее количество строк.

То есть в данном случае мы видим, что у нас всего 16 городов, и в городе с наименьшим населением живет 1012 тыс. человек.

### Группировка DataFrame с уровнями индекса и столбцами

Кроме общей группировки данных по столбцу с помощью функции groupby() Pandas также позволяет сгруппировать данные DataFrame в соответствии с уровнями индекса и столбцами. Это предполагает использование одного или нескольких столбцов в качестве индекса для формирования групп с одновременным выполнением операций агрегирования для определенных столбцов. Что обеспечивает более комплексный анализ данных за счет рассмотрения нескольких уровней организации и соответствующего суммирования данных. Общий синтаксис группировки:

```python
result = df.groupby([уровень_индекса_1, уровень_индекса_2, ...])[столбец_1, столбец_2, ...].agg(функции_агрегирования)
```

Здесь подвыражение `[уровень_индекса_1, уровень_индекса_2, ...]` представляет список столбцов, которые надо использовать в качестве уровней индекса для группировки. Можно указать один или несколько столбцов для формирования иерархического индекса для группировки

Подвыражение `[столбец_1, столбец_2, ...]` представляет список столбцов, для которых надо рассчитать сводную статистику.

И в конце вызывается функция `agg(функции_агрегирования)`, в которую передаются применяемые к выбранным столбцам функции агрегирования. Здесь можно передать словарь имен столбцов в качестве ключей и функции агрегирования в качестве значений.

Рассмотрим следующий пример:

```python
result = df.groupby(['Time zone']).agg({'Population': 'sum', 'City': 'count'})
>>> print(result)
           Population  City
Time zone                  
GMT+3           25807     8
GMT+4            1154     1
GMT+5            4919     4
GMT+6            1101     1
GMT+7            2849     2
```

Здесь идет группировка по столбцу "Time zone", то есть по часовому поясу, с вычислением функции `sum` над столбцом "Population" и функции `count` для столбца "City". То есть рассчитываем суммарное количество населения и общего количество городов в определенном часовом поясе.

В примере выше использовалась группировку по одному уровню индексов - по столбцу "Time zone", но подобным образом мы можем организовать многоуровневую группировку. Например:

```python
data = {
"Employee": ["Tom", "Bob", "Sam", "Alice", "Kate", "Alex", "Mike"],
"Company": ["Microsoft", "Google", "Yandex", "Microsoft", "Google", "Yandex", "Microsoft"],
"Position": ["Developer", "Manager", "Developer", "Manager", "Developer", "Manager", "Developer"],
"Salary": [1637, 1028, 1212, 1548, 1166, 1330, 1198]
}

df = pd.DataFrame(data)
result = df.groupby(["Company", "Position"]).agg({"Salary": "mean", "Employee": "count"})
>>> print(result)
                     Salary  Employee
Company   Position                   
Google    Developer  1166.0         1
          Manager    1028.0         1
Microsoft Developer  1417.5         2
          Manager    1548.0         1
Yandex    Developer  1212.0         1
          Manager    1330.0         1
```

Здесь данные представляют сотрудников некоторых компаний, для которых доступно четыре признака: "Employee" (имя сотрудника), "Company" (компания сотрудника), "Position" (должность) и "Salary" (зарплата).

С помощью выражения `df.groupby(["Company", "Position"])` создаем многоуровневую группировку: сначала данные группируются по столбцу "Company", затем в рамках группы данные далее группируются по столбцу "Position". То есть мы получим конструкцию с несколькими уровнями индексов. Учитывая, что у нас 3 компании и две должности, то теоретически мы можем получить до 3 * 2 = 6 групп. (В реальности их меньше, так как не все компании содержат обе должности).

Далее с помощью вызова `agg({"Salary": "mean", "Employee": "count"})` получаем среднее по зарплате для каждой группы и общее количество человек в каждой группе.

# Группировка и агрегация с пользовательскими функциями

Хотя библиотека Pandas предоставляет набор встроенных функций агрегирования, которые покрывают большинство стандартных потребностей, мы также можем использовать собственные функции. Такой подход позволяет определить собственную логику обобщения и извлечения информации из сгруппированных данных. Рассмотрим небольшой пример:

```python
data = {
    "City": 
    ["Москва", "Санкт-Петербург", "Новосибирск", "Екатеринбург", "Казань", "Красноярск", "Нижний Новгород", "Челябинск", "Уфа", "Краснодар", "Самара", "Ростов-на-Дону", "Омск", "Воронеж", "Пермь", "Волгоград"], 
	"Population": 
[13274, 5653, 1637, 1548, 1330, 1212, 1198, 1177, 1166, 1155, 1154, 1143, 1101, 1042, 1028, 1012],
	"Time zone":["GMT+3","GMT+3","GMT+7","GMT+5","GMT+3","GMT+7","GMT+3","GMT+5","GMT+5",
	"GMT+3","GMT+4","GMT+3","GMT+6","GMT+3","GMT+5","GMT+3"]
}
 
df = pd.DataFrame(data)
 
 
# Определяем кастомную функцию агрегирования
def custom_aggregation (group):
    min_val = min(group["Population"])
    max_val = max(group["Population"])
    sum_val = sum(group["Population"])
    return pd.Series({"Min": min_val, "Max": max_val, "Sum": sum_val})
 
# Группировка данных по столбцу "Time zone" и применение пользовательской функции агрегирования
result = df.groupby("Time zone").apply(custom_aggregation, include_groups=False)
 
>>> print(result)
            Min    Max    Sum
Time zone                    
GMT+3      1012  13274  25807
GMT+4      1154   1154   1154
GMT+5      1028   1548   4919
GMT+6      1101   1101   1101
GMT+7      1212   1637   2849
```

Здесь для каждого часового пояса мы возвращаем максимальное, минимальное и суммарное значение населения, которые живут в городах в данном часовом поясе. Для этого мы определили функцию `custom_aggregation`, которая принимает на вход группу городов и вычисляет значения min_val, max_val и sum_val, применяя соответствующие функции агрегирования, и возвращаем результат в виде объекта `Series`.

Затем мы группируем данные по столбцу "Time zone" (то есть по часовому поясу) с помощью метода `groupby()` и с помощью метода `apply()` применяем функцию `custom_aggregation` к каждой группе городов в определенном часовом поясе. При этом в качестве второго параметра в метод `apply()` передается параметр `include_groups=False`. В итоге функция custom_aggregation получит результат группировки - группы городов и по каждой группе рассчитает минимальное, максимальное значения и сумму.

При этом логика кастомной функции агрегирования может быть сложнее. Например, нам надо получить не просто минимальное и максимальное население, а названия городов:

```python
data = {
    "City": ["Москва", "Санкт-Петербург", "Новосибирск", "Екатеринбург", "Казань", "Красноярск", "Нижний Новгород", "Челябинск", "Уфа", "Краснодар", "Самара", "Ростов-на-Дону", "Омск", "Воронеж", "Пермь", "Волгоград"], 
    "Population": [13274, 5653, 1637, 1548, 1330, 1212, 1198, 1177, 1166, 1155, 1154, 1143, 1101, 1042, 1028, 1012],
    "Time zone": ["GMT+3", "GMT+3","GMT+7","GMT+5","GMT+3","GMT+7","GMT+3","GMT+5","GMT+5","GMT+3","GMT+4","GMT+3","GMT+6","GMT+3","GMT+5","GMT+3"]
}
 
df = pd.DataFrame(data)
 
 
# Определяем кастомную функцию агрегирования
def custom_aggregation (group):
 
    min_val = group[group["Population"] == min(group["Population"])]["City"].values[0]
    max_val = group[group["Population"] == max(group["Population"])]["City"].values[0]
    return pd.Series({"Min": min_val, "Max": max_val})
 
# Группировка данных по столбцу "Time zone" и применение пользовательской функции агрегирования
result = df.groupby("Time zone").apply(custom_aggregation, include_groups=False)
>>> print(result)
                  Min           Max
Time zone                          
GMT+3       Волгоград        Москва
GMT+4          Самара        Самара
GMT+5           Пермь  Екатеринбург
GMT+6            Омск          Омск
GMT+7      Красноярск   Новосибирск
```

Здесь в функции custom_aggregation для получения названия города с минимальным населением в группе (часовом поясе) применяем выражение:

```python
min_val = group[group["Population"] == min(group["Population"])]["City"].values[0]
```

Это выражение можно разбить на ряд подэтапов:

1. `min(group["Population"])`
    
    Получаем минимальное значение по столбцу "Population"
    
2. `group[group["Population"] == min(group["Population"])]`
    
    Находим строку с минимальным значением по столбцу "Population"
    
3. `group[group["Population"] == min(...)]["City"]`
    
    Из строки выбираем столбец "City"
    
4. `group[group["Population"] == min(...)]["City"].values[0]`
    
    Полученный на предыдущем этапе результат представляет объект Series. И чтобы получить собственно название города, обращаемся к коллекции значений этой серии через `values` и с помощью индекса `[0]` берем самое первое значение. Это и будет название города.
    

Получение названия города с максимальным населением в часовом поясе выглядит аналогично.

# Дискретизация и разбиение на интервалы

Дискретизация (Discretization) и разбиение на интервалы (binning) представляют методы группировки непрерывных данных в дискретные интервалы или ячейки. Этот процесс может быть полезен при работе с числовыми данными, которые имеют широкий диапазон значений и требуют категоризации для анализа. Дискретизация помогает упростить данные, уменьшить шум и облегчить анализ за счет создания осмысленных категорий, преобразовать непрерывные данные в категориальные или порядковые.

Для выполнения разбиения на интервалы и дискретизации в библиотеке Pandas применяется функция cut:

```python
pandas.cut(data_series, bins, labels=labels, right=False)
```

Функция принимает следующие параметры:

- `data_series`: это непрерывные числовые данные, которые надо дискретизировать. Это может быть объект Series или массив NumPy.
    
- `bins`: определяет границы или интервалы ячеек. Он может быть указан в виде списка значений, которые представляют границы ячеек. Например, [0, 25, 50, 75, 100] представляет четыре интервала `[0, 25), [25, 50), [50, 75), [75, 100]`.
    
- `labels`: необязательный параметр, который позволяет присваивать метки дискретным интервалам. Его длина должна быть равна количеству интервалов минус один. Эти метки представляют категории или группы, к которым будут отнесены точки данных.
    
- `right`: еще один необязательный параметр, который определяет, открыт или закрыт правый край интервала. По умолчанию он установлен в значение `True`, что делает интервалы закрытыми справа. Установка значения `False` делает интервалы закрытыми слева и открытыми справа.
    

В качестве результата функция `cut()` возвращает новый категориальный объект Series, который содержит метки интервалов для каждой точки данных на основе указанных интервалов и меток.

Для примера рассмотрим следующую программу:

```python
data = {
    "Employees": ["Tom", "Bob", "Sam", "Alice", "Kate", "Alex", "Mike"], 
    "Salary": [1637, 1028, 1212, 1548, 1166, 1330, 1198]
}
 
df = pd.DataFrame(data)
 
 
# Определение интервалов и меток для показателей из столбца "Salary"
salary_bins = [0, 1100, 1500, 2000]
salary_labels = ["Низкий", "Средний", "Высокий"]
 
# Дискретизация зарплаты в столбце Salary
df["Salary_Category"] = pd.cut(df["Salary"], bins=salary_bins, labels=salary_labels, right=False)
 
>>> print(df)
  Employees  Salary Salary_Category
0       Tom    1637         Высокий
1       Bob    1028          Низкий
2       Sam    1212         Средний
3     Alice    1548         Высокий
4      Kate    1166         Средний
5      Alex    1330         Средний
6      Mike    1198         Средний
```

Здесь DataFrame df представляет данные по сотрудникам (столбец "Employees") и их зарплатам (столбец "Salary"). И в данном случае выполняется дискретизация условных зарплат - значений в столбце Salary на отдельные категории (`["Низкий", "Средний", "Высокий"]`). Сначала определяются интервалы и соответствующие метки для классификации зарплат. Список `salary_bins` определяет числовые диапазоны для каждой категории, а список `salary_labels` присваивает метки этим категориям.

В выходных данных каждая строка представляет собой данные работника из оригинального набора данных и соответствующую ему категорию зарплаты. В частности, столбец "Salary_Category" предоставляет категориальную классификацию зарплат. Эта классификация определяется в процессе группировки, где зарплаты группируются в предопределенные диапазоны.

Например, зарплата у сотрудника с именем Tom составляет 1637 единиц, поэтому соответствующая строка попадает в категорию "Высокий". С другой стороны, у сотрудника с именем Bob зарплата 1028 единиц, что относит их к категории "Низкий".

Такое категориальное представление данных позволяет лучше понять общее их качество на основе категорий, упрощая сравнение и анализ без акцентирования внимания на точных числовых различиях.
