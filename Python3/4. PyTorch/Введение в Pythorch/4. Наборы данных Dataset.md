Ключевым компонентом при наборе с моделью являются, конечно же, данные. Данные используются для обучения модели и ее тестировании. Но данные бывают разные - текст, видео, изображения и т.д. В PyTorch, чтобы свести разные типы данных к единому знаменателю, используется специальный объект - набор данных в виде типа torch.utils.data.Dataset.

Для целей обучения и тренировки работы с PyTorch фреймворк предоставляет несколько предустановленных наборов данных. Эти наборы являются подклассами torch.utils.data.Dataset и реализуют функции, специфичные для конкретных данных. Подобные предустановленные наборы данных можно использовать для прототипирования и тестирования моделей машинного обучения.

Предустановленные наборы данных делятся на три типа:

- Наборы данных изображений (наборы данных классификации изображений, наборы данных сегментации изображений, наборы данных оптического потока, стереосоответствие, пары изображений, создание субтитров к изображениям, классификация видео, предсказание видео). Пакет torchvision.datasets
    
- Наборы данных текста (классификация текста, моделирование языка, машинный перевод, разметка последовательностей, вопрос-ответ, обучение без учителя). Пакет torchtext.datasets
    
- Наборы данных аудио (распознавание речи, классификация звуков). Пакет torchaudio.datasets
    

## Загрузка набора данных

В PyTorch загрузка набора данных начинается с определения их местоположения данных: это могут быть файлы на текущем компьютере или на удаленном ресурсе. Затем эти данные организуются в объект Dataset - конструкцию PyTorch, которая стандартизирует форматирование и доступ к данным. Пример подключения:

```python
import torch
from torch.utils.data import Dataset # импортируем Dataset для работы с наборами данных
```

Для примера рассмотрим загрузку одного из встроенных, учебных наборов данных - Fashion-MNIST из пакета torchvision. Fashion-MNIST - это набор изображений из 60 000 обучающих и 10 000 тестовых примеров. Каждый пример представляет изображение серого цвета размером 28×28. На уровне кода Python этот датасет представлен классом FashionMNIST:

```python
class torchvision.datasets.FashionMNIST(
        root: Union[str, Path], 
        train: bool = True, 
        transform: Optional[Callable] = None, 
        target_transform: Optional[Callable] = None, 
        download: bool = False
)
```

Для создания объекта класса применяется ряд параметров:

- `root` (str или pathlib.Path): корневой каталог набора данных, в котором находятся наборы `FashionMNIST/raw/train-images-idx3-ubyte` и `FashionMNIST/raw/t10k-images-idx3-ubyte`.
    
- `train` (bool): если `True`, создаёт набор данных из `train-images-idx3-ubyte`, если `False` - из `t10k-images-idx3-ubyte`.
    
- `transform`: функция преобразования, которая принимает изображение PIL и возвращает преобразованную версию
    
- `target_transform`: функция преобразования, которая принимает целевой объект и преобразует его.
    
- `download` (bool): если `True`, набор данных загружается из интернета и помещается в корневой каталог root. Если набор данных уже загружен, он не загружается повторно.
    

Рассмотрим конкретный пример:

```python
import torch
 
from torchvision import datasets
from torchvision.transforms import ToTensor
 
# набор для обучения
training_data = datasets.FashionMNIST(
    root="data",
    train=True,
    download=True,
    transform=ToTensor()
)
 
# набор для тестирования
test_data = datasets.FashionMNIST(
    root="data",
    train=False,
    download=True,
    transform=ToTensor()
)
```

В данном случае загружаем два набора данных FashionMNIST - по отдельности для обучения и тестирования. Основные моменты:

- `from torchvision import datasets`
    
    импортирует модуль datasets из torchvision, пакета, предоставляющего утилиты для работы с данными изображений в PyTorch.
    
- `from torchvision.transforms import ToTensor`
    
    импортирует преобразование ToTensor, которое преобразует изображения в тензоры PyTorch — основную структуру данных, используемую в PyTorch для вычислений.
    
- `training_data = datasets.FashionMNIST( root="data", train=True, download=True, transform=ToTensor() )`
    
    Создает экземпляр набора данных FashionMNIST, предназначенного для обучения модели. Оно определяет:
    
    - `root="data"`: локальный каталог, в котором будет храниться или из которого будет извлечен набор данных.
        
    - `train=True`: указывает, что этот набор данных будет использоваться для обучения модели.
        
    - `download=True`: если набор данных отсутствует в указанном каталоге, он будет загружен из интернета.
        
    - `transform=ToTensor()`: применяет преобразование `ToTensor` к каждому изображению в наборе данных, преобразуя их из изображений PIL или массивов NumPy в тензоры PyTorch.
        
- `test_data = datasets.FashionMNIST(....)`
    
    Аналогичным образом создаёт экземпляр набора данных FashionMNIST, только теперь для тестирования. Основное отличие заключается в том, что `train=False` указывает, что этот набор данных предназначен для тестирования, а не для обучения.

### Итерация и визуализация набора данных

Итерация и визуализация набора данных в PyTorch включает в себя просмотр каждого элемента набора данных и отображение этого элемента, обычно изображений. Этот процесс помогает лучше понять данные и убедиться в их правильном формате перед началом обучения модели. Для визуализации используем пакет **matplotlib.pyplot**, соответственно его необходимо предварительно установить:

```
pip install matplotlib
```

Для визуализации набора данных определим следующий код:

```python
import torch
 
from torchvision import datasets
from torchvision.transforms import ToTensor
import matplotlib.pyplot as plt
 
# набор для обучения
training_data = datasets.FashionMNIST(
    root="data",
    train=True,
    download=True,
    transform=ToTensor()
)
 
labels_map = {
    0: "T-Shirt",
    1: "Trouser",
    2: "Pullover",
    3: "Dress",
    4: "Coat",
    5: "Sandal",
    6: "Shirt",
    7: "Sneaker",
    8: "Bag",
    9: "Ankle Boot",
}
figure = plt.figure(figsize=(8, 8))
 
cols, rows = 3, 3
 
for i in range(1, cols * rows + 1):
    sample_idx = torch.randint(len(training_data), size=(1,)).item()
    img, label = training_data[sample_idx]
    figure.add_subplot(rows, cols, i)
    plt.title(labels_map[label])
    plt.axis("off")
    plt.imshow(img.squeeze(), cmap="gray")
 
# отображаем набор данных
plt.show()
```

Разберем пошагово основные аспекты кода:

- Определение меток и подготовка графика

```python
labels_map = {
    # ... определение меток ...
}
figure = plt.figure(figsize=(8, 8))
cols, rows = 3, 3
```

- Здесь выполняются следующие действия:
    
    1. Создается словарь `labels_map`, который сопоставляет числовые метки (от 0 до 9) из набора данных с соответствующими текстовыми названиями категорий (например, 0: "T-Shirt", 9: "Ankle Boot").
        
    2. Далее создается новая фигура Matplotlib (`figure`) с размером 8x8, которая будет служить контейнером для отображения изображений.
        
    3. Определяются переменные `cols` и `rows` (3 и 3), устанавливая размер сетки для отображения изображений (3x3 = 9 изображений).
        
- На следующем шаге выполняется визуализация случайных образцов

```python
for i in range(1, cols * rows + 1):
    sample_idx = torch.randint(len(training_data), size=(1,)).item()
    img, label = training_data[sample_idx]
    figure.add_subplot(rows, cols, i)
    plt.title(labels_map[label])
    plt.axis("off")
    plt.imshow(img.squeeze(), cmap="gray")
```

- Цикл выполняет следующие действия 9 раз (для 3x3 ячеек):
    
    1. `sample_idx = torch.randint(...)`: случайным образом выбирается индекс (`sample_idx`) из обучающего набора данных.
        
    2. `img, label = training_data[sample_idx]`: по выбранному индексу из набора данных извлекаются изображение (`img`) и его метка (`label`).
        
    3. `figure.add_subplot(rows, cols, i)`: добавляется новая область к фигуре в текущую позицию сетки.
        
    4. `plt.title(labels_map[label])`: устанавливается заголовок с помощью текстовой метки из `labels_map`.
        
    5. `plt.axis("off")`: скрываются оси x и y, чтобы отображалось только изображение.
        
    6. `plt.imshow(img.squeeze(), cmap="gray")`: отображается изображение на основе данных, где `.squeeze()` убирает лишнее измерение канала (изображения Fashion-MNIST имеют размер `[1, 28, 28]`, а `imshow` ожидает `[28, 28]`). А `cmap="gray"` указывает Matplotlib отображать изображение в сером цвете.
        
- Последний шаг - отображение окна

```python
# отображаем набор данных
plt.show()
```

Таким образом, мы отображаем изображение с 9 случайно выбранными образцами из набора данных Fashion-MNIST, каждый из которых подписан своей категорией. И после запуска программы мы увидим визуализированные данные:
