
Одной из фундаментальных концепций машинного обучения являются градиенты. Если кратко, то градиенты нужны, чтобы понять, в каком направлении и насколько сильно нужно изменить параметры модели (тензоры), чтобы она стала лучше выполнять свою задачу и допускала меньше ошибок. После получения градиента его используют в алгоритме обратного распространения (**back propagation**). Этот алгоритм предусматривает, что параметры (веса модели) корректируются в соответствии с градиентом функции потерь (**loss function**) относительно заданного параметра до тех пор, пока потери не будут минимизированы.

Чтобы лучше понять их сущность, возьмем следующую аналогии - "спуск с горы в тумане". Представьте, что вы стоите на склоне холма в густом тумане. Ваша цель - спуститься в самую низкую точку - подножье холма. Из-за тумана вы не видите весь путь, но можете ногой почувствовать уклон прямо под собой. И в данном случае при спуске с горы мы будем иметь следующие условные параметры:

- Ваша позиция: это текущие значения параметров модели (тензоры весов и смещений).
    
- Высота в данной точке: это функция потерь (Loss function), которая измеряет, насколько плохи предсказания модели. Чем выше Loss, тем хуже модель.
    
- Наклон (градиент) под вашими ногами: это и есть градиент. Он указывает направление самого крутого подъема.
    

Чтобы спуститься (уменьшить ошибку), надо сделать шаг в противоположном направлении от градиента. Градиент - это своего рода компас в тумане.

Если же говорить про градиент в математическом смысле, то это вектор частных производных.

Для функции потерь `L` (Loss function), которая зависит от миллионов параметров тензора `W`, градиент `∇L(W)` - это вектор, где каждый элемент говорит: "Если я немного увеличу именно этот параметр `w_i`, насколько сильно изменится общая ошибка `L`?" Положительная производная означает, что увеличение параметра увеличит ошибку (плохо). Отрицательная - что увеличение параметра уменьшит ошибку (хорошо).

Градиент указывает направление наискорейшего роста функции. Нам же нужно двигаться в обратном направлении.

А сам процесс обучения нейронной сети обычно предполагает итеративное повторение трех шагов:

1. Прямой проход (Forward Pass): подаем данные на вход сети, получаем предсказания. Сравниваем с правильными ответами и вычисляем функцию потерь `L` (одно число, которое показывает, "насколько плох" результат).
    
2. Обратное распространение (Backpropagation): это ключевой этап, на котором вычисляется градиент функции потерь по каждому параметру модели (`∂L/∂W` для каждого тензора `W`). Это и есть "градиенты для тензоров". Они имеют ту же размерность, что и сами тензоры-параметры.
    
3. Обновление параметров (Update): слегка изменяем каждый параметр в направлении, противоположном его градиенту.

```
W_новый = W_старый - α * ∂L/∂W
```

Где `α` - скорость обучения (learning rate), небольшое число, которое определяет размер шага.

## Автоматическое дифференцирование

Для упрощения вычисления градиентов в PyTorch есть встроенный механизм - автоматическое дифференцирование в виде функции torch.autograd. Autograd хранит данные (тензоры) и все выполненные операции (вместе с полученными новыми тензорами) в DAG-графе (directed acyclic graph - ориентированный/направленный ациклический граф), который состоит из объектов Function. В этом DAG-графе листья являются входными тензорами, корни - выходными. Прослеживая этот граф от корней к листьям, можно по цепочке автоматически вычислять градиенты.

Autograd выполняет два прохода: прямой (forward pass) и обратный (backward pass). При прямом проходе Autograd выполняет две задачи одновременно:

- выполняет запрошенную операцию для вычисления результирующего тензора
    
- сохраняет функцию градиента операции в DAG.
    

При обратном проходе Autograd:

- вычисляет градиенты, используя функции, ссылки на котрые хранятся в тензоре в свойстве `.grad_fn`
    
- вычисленные градиенты тензора сохраняются в его атрибуте `grad`
    
- используя правило цепочки (chain rule), вычисленные градиенты распространяются вплоть до тензоров-листьев.
    

### require_grad

Прежде всего, чтобы для тензора вычислялись градиенты, необходимо установить его свойство require_grad в `True`. Это свойство можно с помощью одноименного параметра функции создания тензора:

```python
x = torch.ones(4, requires_grad = True)
print(x)    # tensor([1., 1., 1., 1.], requires_grad=True)
print(x.requires_grad)  # True
 
y = torch.tensor([1., 2., 3., 4.], requires_grad = True)
print(y)    # tensor([1., 2., 3., 4.], requires_grad=True)
print(y.requires_grad)  # True
```

В качестве альтернативы для переключения этого свойства можно использовать метод `requires_grad_()`:

```python
x = torch.ones(4)
print(x)    # tensor([1., 1., 1., 1.])
print(x.requires_grad)  # False
 
x.requires_grad_(True)
print(x)    # tensor([1., 1., 1., 1.], requires_grad=True)
print(x.requires_grad)  # True
```

Стоит отметить, что только для тензоров с числами с плавающей точкой и комплексных типов можно устанавливать градиенты.

### grad_fn

Функция, которая применяется к тензорам для построения вычислительного графа, фактически является объектом класса Function. Этот объект умеет вычислять функцию в прямом направлении, а также вычислять ее производную на этапе обратного распространения (backward propagation). Ссылка на функцию обратного распространения хранится в свойстве grad_fn тензора:

```python
# создаем тензор с requires_grad=True, чтобы указать, что мы хотим вычислить для него градиент
x = torch.tensor([1.5], requires_grad=True)
 
# выполняем операции на тензоре
y = x ** 2 + 4 * x + 11
 
print(y)    # tensor([19.2500], grad_fn=<AddBackward0>)
 
print(x.grad_fn)        # None
print(y.grad_fn)        # Что-то наподобие <AddBackward0 object at 0x7291aef02080>
```

Здесь мы определеняем функцию `y`

```python
y = x ** 2 + 4 * x + 11
```

С точки зрения математики это уравнение:

`y = x^2 + 4x + 11`

Эта функция осуществляет так называемый прямой проход (Forward Pass). Так как для `x` установлен флаг `requires_grad=True`, PyTorch строит динамический вычислительный граф (computational graph). Он запоминает, какие операции были произведены, чтобы получить `y`.

При выводе мы увидим, что `y` также представляет тензор, у которого установлено свойство `grad_fn` - хранит ссылку на функцию обратного распространения ошибки. Через эту функцию и будет вычисляться градиент относительно тензора `x`

### Обратное распространение и вычисление ошибки

Для выполнения обратного распространения и вычисления градиента применяется функция `backward()`:

```python
# создаем тензор с requires_grad=True, чтобы указать, что мы хотим вычислить для него градиент
x = torch.tensor([1.5], requires_grad=True)
 
# выполняем операции на тензоре
y = x ** 2 + 4 * x + 11
 
# вычисляем градиент для y в отношении x
y.backward()
 
# градиент y в отношении x сохранен в x.grad
print(x.grad)   # tensor([7.])
```

Подробно рассмотрим, что происходит на каждом этапе:

1. Создание тензора с отслеживанием градиента

```python 
x = torch.tensor([1.5], requires_grad=True)
```

   Здесь мы создаем тензор `x` с параметром `require_grad=True`, который указывает PyTorch отслеживать все операции, производимые с `x`. Это необходимо для того, чтобы позже "прокрутить пленку назад" и вычислить градиенты (производные).

2. Определение функции (Прямой проход / Forward Pass)

```python
y = x ** 2 + 4 * x + 11
```
