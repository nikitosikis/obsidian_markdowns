# Что такое Алгоритм

Алгоритм - это набор инструкций для решения задачи.
Алгоритм должен обладать следующими свойствами:
1. Четко определенные задача, вход, выход алгоритма
2. Шаги алгоритма должны идти в четко определенном порядке
3. Шаги должны быть четко определены
4. Алгоритм должен давать результат
5. Алгоритм должен исполнятся за конечное количество времени

# Сложность алгоритма
Большое O - теоретическое определение сложности алгоритма как функции размера, т.е. насколько долго алгоритм исполняется в зависимости от размера входных данных

Примеры:
O(n)
O(n^2)
O(log(n))
O(1)
O(n log(n))
O(x^n)
O(n!)

## Алгоритмы поиска
### Линейный поиск
В данном случае линейный поиск описан применимо к любому списку
Линейный поиск состоит из 4 пунктов:
1. Начало итераций в начале списка
2. Сравнение текущего элемента с таргетом
3. продвижение к следующему элементу
4. Конец списка
Алгоритм завершает свою работу если найден нужный таргет или достигнут конец списка
(В представленной далее реализации в качестве результата работы алгоритма мы получаем True если искомый таргет есть в списке и False если нет)
Сложность алгоритма - O(n)

```python

def linear_search(input_list: list, target) -> bool:
	for i in input_list:
		if target == i:
			return True
	return False

linear_search([1,2,3], 4) #False
linear_search([1,2,3], 3) #True
```

### Бинарный поиск
Бинарный поиск можно использовать только для отсортированных списков.
В данной итерации алгоритма реализован поиск по отсортированному списку чисел
Бинарный поиск состоит из 3 пунктов:
1. Задание первого и последнего индекса списка
2. Проверка таргета по среднему значению
3. Смещение таргета в центр усеченного списка
Сложность алгоритма O(log(n))

```python

def binary_search(input_list: list, target: int) -> bool:
	first_ind = 0
	last_ind = len(input_list) - 1
	
	while first_ind <= last_ind:
		midpoint = (first_ind + last_ind) // 2
		if input_list[midpoint] == target:
			return True
		elif input_list[midpoint] < target:
			first_ind = midpoint + 1
		elif input_list[midpoint] > target:
			last_ind = midpoint - 1
	
	return False
	
binary_search([-3,4,5,10,12], 11) # False
binary_search([-3,4,5,10,12], 12) # True
```

### Рекурсивный бинарный поиск

Рекурсивный бинарный поиск можно использовать только для отсортированных списков.

В данной реализации алгоритма выполняется поиск по отсортированному списку чисел.  
Главное отличие от обычного бинарного поиска в том, что вместо цикла `while` функция вызывает сама себя, каждый раз уменьшая область поиска.

Рекурсивный бинарный поиск состоит из 4 пунктов:

1. Задание левой и правой границы поиска
    
2. Проверка среднего элемента списка
    
3. Если средний элемент равен таргету — вернуть `True`
    
4. Если таргет меньше или больше среднего элемента — рекурсивно вызвать функцию для левой или правой части списка
    

Алгоритм завершает свою работу, если найден нужный таргет или если левая граница стала больше правой, то есть область поиска закончилась.

Сложность алгоритма — `O(log(n))`

Дополнительная память — `O(log(n))`, потому что при рекурсии создаются новые вызовы функции.

```python

def recursive_binary_search(input_list: list, target: int, first_ind: int, last_ind: int) -> bool:
    if first_ind > last_ind:
        return False

    midpoint = (first_ind + last_ind) // 2

    if input_list[midpoint] == target:
        return True
    elif input_list[midpoint] < target:
        return recursive_binary_search(input_list, target, midpoint + 1, last_ind)
    elif input_list[midpoint] > target:
        return recursive_binary_search(input_list, target, first_ind, midpoint - 1)


recursive_binary_search([-3, 4, 5, 10, 12], 11, 0, 4) # False
recursive_binary_search([-3, 4, 5, 10, 12], 12, 0, 4) # True
```
